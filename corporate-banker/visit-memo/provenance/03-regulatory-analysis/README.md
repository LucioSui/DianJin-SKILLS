# 03 - 监管合规分析

## 输入

- 《商业银行授信工作尽职指引》(银监发〔2004〕51号)
- 《贷款风险分类指引》(银监发〔2007〕54号)
- 内部贷前调查管理规范

## 过程

### 监管要求提取

| 监管文件 | 核心要求 | 映射到Skill功能 |
|---------|---------|----------------|
| 授信工作尽职指引 | 贷前调查必须全面、真实、准确 | 7个结构化维度定义 |
| 贷款风险分类指引 | 风险信号必须及时识别和报告 | 步骤4风险信号扫描 |
| 内部贷前调查规范 | 拜访记录须标注信息来源和时间 | 数据溯源要求 |

### 红线定义

基于监管要求,定义3条一票否决红线:
- R1: 用户要求记录涉嫌财务造假的信息(须提示合规风险)
- R2: 用户要求隐瞒已知风险信号(须提示如实记录义务)
- R3: 用户要求伪造现场观察数据(须拒绝并记录合规事件)

### 合规检查清单

- [x] 事实与判断分离(Constraints第1条)
- [x] 风险信号单独标注(步骤4)
- [x] 数据溯源(每条信息标注来源)
- [x] 禁止越权建议(Constraints第7条)
- [x] 红线执行强制(Constraints第9条)

## 输出

- 监管要求映射表(3条)
- 红线定义(3条R1-R3)
- 合规检查清单(5项)
