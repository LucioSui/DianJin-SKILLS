# 04 - Skill合成与迭代

## 输入

- 原SKILL.md(407行)

- 监管要求(来自03-regulatory-analysis)

## 过程

### 核心业务逻辑保护清单

改造过程中严格保护以下9条核心业务规则,未做任何修改:

1. **7个结构化维度**: 企业概况、财务线索、上下游关系、担保物与资产、风险信号、人物印象、拜访成果
2. **事实与判断分离**: 客观描述用正常文字,客户经理判断用括号标注"(CM判断:...)"
3. **增量合并机制**: 新信息与已有记录合并,不覆盖、不丢失
4. **风险前置**: 风险信号在记录末尾单独标注⚠️
5. **追问克制**: 每轮最多追问1-2个问题,优先追问风险维度
6. **模式灵活**: 用户说"先不保存"时只展示不保存
7. **数据溯源**: 每条记录须标注信息来源和时间戳
8. **3条金融合规红线**: R1(财务造假)、R2(隐瞒风险)、R3(伪造数据)
9. **4种降级策略**: ECIF不可用、历史记录不可用、风险预警不可用、语音转文本不可用

### 关键创新点

1. **步骤0先读后写机制**: 强制数据确认,防止数据质量问题导致后续流程失败
2. **模式D交互模式**: 对话阶段(开始→中间→结束)清晰标注,上下文管理规则明确
3. **防偷懒指令**: 步骤0/4/5增加"不得跳过"、"不得追问宽泛问题"等对抗指令
4. **结构化输出**: 7章节表格+下游可解析字段,确保 credit-due-diligence 和 submit-credit-application Skill 可靠提取
5. **多轮对话限制**: 单轮拜访记录最多20轮增量更新,防止上下文溢出
