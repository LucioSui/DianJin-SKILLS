# 02 - 专家访谈与经验萃取

## 输入

- 8位资深对公客户经理访谈记录
- 风险经理风险识别要求
- 历史拜访记录案例分析(2024-2025年)

## 过程

### 专家经验萃取

| 专家经验 | 来源 | 应用到Skill |
|---------|------|------------|
| 事实与判断必须分离 | 风险经理反馈 | Constraints第1条,输出要求标注来源 |
| 风险信号不能淹没在正文中 | 审批退回案例 | 步骤4强制风险扫描,末尾单独标注⚠️ |
| 追问必须精准,不能宽泛 | 客户经理踩坑 | 步骤5强制指令,禁止"还有什么要补充的" |
| 增量更新不能覆盖历史 | 数据丢失案例 | 步骤2增量合并规则,标注"新增:"/"已更正:" |
| 口述信息与现场观察要区分 | 尽调报告退回 | 每条信息标注(企业方口述)/(现场观察)/(CM判断) |

### 踩坑点识别

1. **口述信息与现场观察混淆**:将主观判断误标为客观事实
   - **解决**: 所有信息必须标注来源

2. **风险信号被淹没**:重要风险信号混杂在财务线索段落中
   - **解决**: 每次整理后必须扫描风险信号维度,在末尾单独列出

3. **增量更新覆盖历史**:新信息替换了旧信息,导致数据丢失
   - **解决**: 任何更新必须保留原值,新增内容标注"新增:",修正内容标注"已更正:"

4. **追问过于宽泛**:追问"还有什么要补充的吗",用户不知从何说起
   - **解决**: 使用追问话术参考表,精准指向缺失维度

### 结构化维度优化

基于专家经验,优化7个维度的字段定义:
- 企业概况: 厂区面积、员工规模、生产线、经营活跃度(可现场观察)
- 财务线索: 营收、利润、资金、负债、票据(需标注口述/观察)
- 上下游关系: 客户集中度、供应商、行业地位、话语权
- 担保物与资产: 不动产、设备、其他资产、已有抵押
- 风险信号: 经营类、财务类、关联方、法律合规类(4类)
- 人物印象: 实控人、财务负责人、其他关键人
- 本次拜访成果: 目标达成、资料清单、承诺约定

## 输出

- 专家经验5条
- 踩坑点识别4个
- 结构化维度优化方案
