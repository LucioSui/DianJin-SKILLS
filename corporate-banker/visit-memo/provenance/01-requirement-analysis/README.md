# 01 - 需求分析与框架设计

## 输入

- 原SKILL.md(407行,7步Workflow)
- 客户经理现场拜访记录需求
- 贷前尽调规范要求

## 过程

### 需求提取

| 需求项 | 来源 | 应用到Skill |
|--------|------|------------|
| 现场口述实时记录 | 客户经理反馈 | 步骤1首次输入记录 |
| 多轮增量合并 | 拜访场景特点 | 步骤2后续增量合并 |
| 风险信号识别 | 风险经理要求 | 步骤4风险信号扫描 |
| 追问策略 | 专家经验 | 步骤5缺失追问 |
| 不保存模式 | 灵活场景需求 | 步骤6不保存模式 |

### 结构化维度设计

基于贷前尽调规范,定义7个结构化维度:
1. 企业概况(现场可见事实)
2. 财务线索(口述与现场佐证)
3. 上下游关系
4. 担保物与资产
5. 风险信号识别
6. 人物印象
7. 本次拜访成果(后期补充)

### 交互模式判定

**判定结果**: 模式 D - 对话辅助型

**判定依据**:
- 实时交互、多轮对话(现场口述→记录→追问→补充)
- 上下文持续累积(增量合并机制)
- 支持多轮更新(步骤2/3/4/6/7都是对话交互)
- 需要增加 Interaction Pattern 和 Context Management 章节

## 输出

- 交互模式判定报告: 模式D-对话辅助型
- 结构化维度定义: 7个维度
- provenance/ 命名方案: 01-requirement-analysis → 02-expert-interview → 03-regulatory-analysis → 04-skill-synthesis
