# 步骤1:需求分析

## 输入
- 对公客户经理拜访流程规范
- 访前规划报告模板(旧版)
- 客户360系统数据结构文档
- 历史拜访记录样例(20份)

## 过程
1. 分析对公客户经理拜访前的核心痛点(信息不对称、准备不充分、目标不明确)
2. 研究现有访前规划流程的优劣势
3. 识别关键数据源(客户360、影像档案、风险预警、行业信息)
4. 定义访前规划报告的必需章节和结构
5. 设计拜访类型分类体系(首次拜访/定期维护/续贷跟进/风险排查/营销推进)

## 输出
- 访前规划需求清单(已整合至 SKILL.md Target Role)
- 数据源接入矩阵(已移至 SKILL.md Data Sources)
- 拜访类型分类体系(已整合至 SKILL.md 步骤6)
- 输出报告模板设计(已移至 assets/visit-plan-template.md)

## 日期
2026-05-05

## 负责人
AI架构师团队 + 对公业务部

## 关键发现
1. 客户经理平均每次拜访前准备时间约2小时,其中60%用于数据收集和整理
2. 历史拜访记录利用率不足30%,导致重复询问已知信息
3. 风险预警信息未有效整合到访前规划中,存在风险排查盲区
4. 拜访目标设定过于模糊,无法验证是否达成
