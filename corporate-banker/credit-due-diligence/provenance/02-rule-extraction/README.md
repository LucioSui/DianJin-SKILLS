# 步骤2：规则提取与结构化

## 输入
- 步骤1输出的监管要求清单
- 行内风险评级政策（references/risk-rating-policy.md）
- 行业分析指南（references/industry-analysis-guide.md）
- 历史尽调报告案例（10份）

## 过程
1. 将监管要求转化为可执行的规则（如"DSCR ≥ 1.2为达标"）
2. 提取财务指标阈值（流动比率、有息负债率、净现比等）
3. 定义关联风险判断标准（其他应收款/净资产 > 20%等）
4. 结构化7条一票否决条件（D1-D7），明确触发依据和降级结果
5. 建立风险等级划分标准（AAA-C九级分类）

## 输出
- 财务指标阈值表（已移至 references/risk-rating-policy.md）
- 关联风险判断标准（已整合至 SKILL.md 步骤4）
- 一票否决条件清单（D1-D7，已整合至 SKILL.md）
- 风险等级与授信策略映射表（references/risk-rating-policy.md）

## 关键决策
1. 采用九级风险分类（AAA-C），而非简单的"高/中/低"三级分类
2. D1-D4为核心红线，触发后必须出具否决意见
3. D5-D7为警告红线，触发后可有条件推荐但须显著标注
4. 所有阈值必须可量化，禁止模糊表述（如"较高"、"偏低"）
