## 知识源清单

### 知识源1：监管文件与制度
- **来源**：《商业银行授信工作尽职指引》、《贷款风险分类指引》、《商业银行法》第35条、《征信业管理条例》、《流动资金贷款管理暂行办法》
- **萃取方法**：法规研读与规则提取
- **萃取日期**：2026-05-05
- **萃取人**：AI架构师团队 + 风险管理部
- **关键内容**：
  - 监管要求尽调必须覆盖企业基本面、财务状况、经营真实性、主体资信四大维度
  - 强调"实质重于形式"原则，要求交叉验证
  - 明确禁止向淘汰类产业新增授信
  - 要求建立完整的审计追踪机制
  - 征信查询需客户授权，严禁未授权查询

### 知识源2：行内风险评级政策
- **来源**：风险管理部《风险等级划分标准》（references/risk-rating-policy.md）
- **萃取方法**：制度文件分析与阈值提取
- **萃取日期**：2026-05-05
- **萃取人**：AI架构师团队 + 信贷审批部
- **关键内容**：
  - 九级风险分类体系（AAA-C）
  - 定量指标权重60%（DSCR、有息负债率、净现比、流动比率）
  - 定性指标权重40%（行业风险、经营风险、管理风险、关联风险）
  - 7条一票否决条件（D1-D7）及降级规则
  - D1-D4触发后必须出具否决意见，D5-D7可有条件推荐

### 知识源3：行业信贷政策与基准数据
- **来源**：发改委《产业结构调整指导目录》、行内行业研究数据库（references/industry-analysis-guide.md）
- **萃取方法**：政策分析与数据提取
- **萃取日期**：2026-05-05
- **萃取人**：AI架构师团队 + 行业研究部
- **关键内容**：
  - 行业生命周期判断框架（初创/成长/成熟/衰退）
  - 产业准入分类（鼓励类/允许类/限制类/淘汰类）
  - 制造业、批发零售业、建筑业财务基准数据（毛利率、净利率、流动比率等）
  - 行业税负率参考范围（制造业增值税1%-3%）
  - 行业信贷政策要点（积极支持类/审慎介入类/限制禁止类）

### 知识源4：历史尽调报告案例
- **来源**：行内信贷系统历史尽调报告（10份，已脱敏）
- **萃取方法**：案例分析与最佳实践提取
- **萃取日期**：2026-05-05
- **萃取人**：AI架构师团队
- **关键内容**：
  - 优秀尽调报告的结构特征（9章全覆盖、量化指标、数据来源标注）
  - 常见缺陷（经营真实性验证不完整、关联风险被忽视、授信建议缺乏测算依据）
  - 红线触发案例（逃废债记录、经营现金流连续为负、存贷双高）
  - 报告输出格式规范（表格化、结构化、下游可解析）

## 生产管线

### 步骤1：监管文件研读（01-regulatory-review）
- **输入**：5部核心监管文件 + 发改委产业目录
- **过程**：逐条研读监管文件，提取尽职调查相关的核心要求，识别监管红线和禁止性行为，分析监管对贷前调查的强制性要求，对照行内信贷政策确认合规边界
- **输出**：监管要求清单、合规红线定义（7条一票否决条件的监管依据）、尽调流程框架（9步流程的监管合规性验证）
- **日期**：2026-05-05
- **负责人**：AI架构师团队 + 风险管理部
- **详细记录**：见 [provenance/01-regulatory-review/README.md](provenance/01-regulatory-review/README.md)

### 步骤2：规则提取与结构化（02-rule-extraction）
- **输入**：步骤1输出的监管要求清单 + 行内风险评级政策 + 行业分析指南 + 10份历史尽调报告
- **过程**：将监管要求转化为可执行的规则（如"DSCR ≥ 1.2为达标"），提取财务指标阈值，定义关联风险判断标准，结构化7条一票否决条件（D1-D7），建立风险等级划分标准（AAA-C九级分类）
- **输出**：财务指标阈值表、关联风险判断标准、一票否决条件清单（D1-D7）、风险等级与授信策略映射表
- **日期**：2026-05-05
- **负责人**：AI架构师团队 + 信贷审批部
- **详细记录**：见 [provenance/02-rule-extraction/README.md](provenance/02-rule-extraction/README.md)

### 步骤3：结构化设计（03-structured-design）
- **输入**：步骤2输出的规则清单和阈值表 + 现有 SKILL.md（v1.0.0）+ METHODOLOGY.md + skill-refactor 改造操作指南
- **过程**：对照 METHODOLOGY.md 的12项核心标准评估现有 SKILL.md 差异，重构 Frontmatter 增加编排字段，将 Workflow 从9步扩展为10步（增加步骤0），为每个步骤添加门控条件，增强 Output Format 结构化程度，创建验证脚本，设计测试用例
- **输出**：新版 SKILL.md（v2.0.0）设计方案、验证脚本设计方案、测试用例集设计方案
- **日期**：2026-05-05
- **负责人**：AI架构师团队
- **详细记录**：见 [provenance/03-structured-design/README.md](provenance/03-structured-design/README.md)

### 步骤4：Skill 合成与迭代（04-skill-synthesis）
- **输入**：步骤3输出的结构化设计方案 + 现有 SKILL.md（v1.0.0）业务逻辑保护清单 + METHODOLOGY.md 快速检查清单
- **过程**：按照设计方案逐项修改 SKILL.md，保留原有15条核心业务规则，重构 Frontmatter（version: 1.0.0 → 2.0.0），重写 Workflow 增加步骤0和门控条件，优化 Output Format，更新 Constraints，校验 SKILL.md 行数，创建 provenance/ 目录和 PROVENANCE.md，更新测试用例
- **输出**：新版 SKILL.md（v2.0.0，~450行）、PROVENANCE.md（~150行）、provenance/ 目录（4个子目录）、更新后的 tests/test_cases.yaml（12个用例）
- **日期**：2026-05-05
- **负责人**：AI架构师团队
- **详细记录**：见 [provenance/04-skill-synthesis/README.md](provenance/04-skill-synthesis/README.md)
