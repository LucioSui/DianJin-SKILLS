# 步骤1:需求分析

## 输入
- 对公客户经理产品路演流程规范
- 产品推介方案模板(旧版)
- 客户360系统数据结构文档
- 历史路演方案样例(20份)

## 过程
1. 分析对公客户经理产品路演的核心痛点(产品堆砌、需求不匹配、话术通用化)
2. 研究现有产品推介流程的优劣势
3. 识别关键数据源(客户360、产品管理系统、竞争行信息、产品谱系)
4. 定义产品路演方案的必需章节和结构
5. 设计路演类型分类体系(专项融资/存款提升/综合金融/特定产品推介)

## 输出
- 产品路演需求清单(已整合至 SKILL.md Target Role)
- 数据源接入矩阵(已移至 SKILL.md Data Sources)
- 路演类型分类体系(已整合至 SKILL.md 差异化方案策略)
- 输出报告模板设计(已移至 assets/roadshow-plan-template.md)

## 日期
2026-05-05

## 负责人
AI架构师团队 + 对公业务部 + 产品管理部

## 关键发现
1. 客户经理平均每次路演准备时间约3小时,主要耗时在产品匹配和竞品分析
2. 成功路演的核心要素:需求诊断准确(占比40%)、产品匹配精准(占比30%)、话术个性化(占比20%)
3. 失败路演的常见原因:产品堆砌(占比50%)、需求不匹配(占比30%)、竞品分析虚假(占比20%)
4. 需要建立"需求驱动→产品匹配→竞品对比→收益测算→话术设计"的标准流程

## 监管要求映射
- 《商业银行产品营销管理办法》第12条:产品推介须基于客户真实需求
- 《商业银行客户关系管理指引》第8条:客户营销须体现个性化特征
