# 步骤3:监管政策分析

## 输入
- 《商业银行产品营销管理办法》
- 《商业银行客户关系管理指引》
- 《商业银行理财业务监督管理办法》
- 《个人信息保护法》
- 产品推介合规要求

## 过程
1. 研读监管文件,提取与产品营销和推介相关的核心要求
2. 识别产品推介的合规边界和禁止性行为
3. 分析《商业银行理财业务监督管理办法》对理财/保险产品推介的要求
4. 研究竞品对比的合规要求(不得贬低竞争行、不得捏造数据)
5. 建立3条金融合规红线(R1-R3),明确触发条件和处理措施

## 输出
- 3条金融合规红线定义(R1-R3,已整合至 SKILL.md Constraints)
- 产品推介合规指南(已整合至 SKILL.md Constraints)
- 数据脱敏规则(5条,已整合至 SKILL.md Data Sources)
- 理财/保险产品风险提示要求(已整合至 SKILL.md Constraints)

## 监管要求映射
1. **《商业银行产品营销管理办法》第12条**:产品推介须基于客户真实需求,不得强制捆绑销售
   - 对应约束:需求诊断优先(Constraint #3)
   
2. **《商业银行理财业务监督管理办法》第25条**:理财/保险产品推介须包含风险提示,不得承诺保本保收益
   - 对应红线:R1(禁止承诺保本保收益)
   - 对应约束:合规边界(Constraint #5)

3. **《商业银行客户关系管理指引》第15条**:客户营销须遵守公平竞争原则,不得贬低竞争行
   - 对应红线:R2(禁止捏造竞争行劣势数据)
   - 对应约束:竞争敏感性(Constraint #4)

4. **《个人信息保护法》第13条**:客户信息使用须符合最小必要原则
   - 对应约束:数据脱敏规则(Data Sources)

## 合规检查清单
- [ ] 产品推介是否基于客户真实需求(需求诊断章节是否存在)
- [ ] 理财/保险产品是否包含风险提示
- [ ] 竞品对比是否使用真实市场数据
- [ ] 收益测算是否标注假设条件
- [ ] 是否包含免责声明
