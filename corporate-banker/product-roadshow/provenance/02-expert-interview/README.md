# 步骤2:专家访谈与经验萃取

## 输入
- 5位资深对公客户经理访谈记录
- 5位产品经理产品推介经验
- 30份历史路演方案(涵盖不同路演类型)
- 成功路演案例(10份)和失败路演案例(10份)

## 过程
1. 访谈对公客户经理,了解产品路演的实际痛点和最佳实践
2. 萃取产品经理关注的核心要素(产品适配性、收益测算准确性、竞品对比真实性)
3. 分析历史路演方案的优劣势,提炼成功要素
4. 识别常见踩坑点(产品堆砌、竞品对比虚假、收益测算假设不透明、话术通用化)
5. 建立产品推荐克制原则(核心≤3,次要≤2)

## 输出
- 产品路演核心信息清单(已整合至 SKILL.md 步骤1-5)
- 产品推荐克制原则(已整合至 SKILL.md 步骤7)
- 踩坑记录(6条 Gotchas,已整合至 SKILL.md)
- 产品谱系指南(已移至 references/product-catalog.md)
- 差异化方案策略(已整合至 SKILL.md)

## 专家经验萃取结果
1. **需求诊断优先**:成功路演的第一步永远是准确诊断客户需求,而非直接介绍产品
2. **产品数量克制**:核心推荐产品不超过3个,客户才能记住重点
3. **竞品对比真实**:必须使用市场真实行情区间,客户往往会向竞争行核实
4. **收益测算透明**:所有测算必须标注假设条件,客户会自行验证
5. **话术个性化**:必须嵌入客户行业名称、具体数字、真实案例,否则客户感觉不被重视

## 踩坑点识别
1. 产品堆砌→客户不知道重点是什么
2. 竞品对比虚假→客户核实后失去信任
3. 收益测算假设不透明→客户无法验证,怀疑数据真实性
4. 话术通用化→客户感觉不被重视,认为客户经理未做功课
5. 忽略产品持有现状→推荐了客户已持有的产品,显得不专业
6. 数据时效性未标注→客户无法判断数据是否最新

## 验证方法
- 对比成功路演方案和失败路演方案的产品数量分布
- 分析客户反馈中关于"产品太多记不住"的频率
- 统计竞品对比数据被客户核实的准确率
