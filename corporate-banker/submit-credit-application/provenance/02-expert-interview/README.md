# 02 - 专家访谈与经验萃取

## 输入

- 5位资深对公客户经理访谈记录
- 信贷审批官审核意见汇总
- 历史案件退回案例分析(2024-2025年)

## 过程

### 专家经验萃取

| 专家经验 | 来源 | 应用到Skill |
|---------|------|------------|
| 材料数量充足不代表质量达标 | 信贷审批官反馈 | 步骤2强制要求读取关键材料内容 |
| "好的"不等于确认提交 | 客户经理踩坑案例 | 步骤5确认机制强化(必须明确说"提交") |
| 案件冲突查询超时必须人工确认 | 系统故障案例 | 步骤1门控分支增加⚠️超时处理 |
| 续贷和追加授信容易混淆 | 客户经理反馈 | 步骤4增加案件类型确认表 |
| 尽调报告必须包含经营和财务分析 | 审批退回案例 | references/material-checklist.md评估要点 |

### 踩坑点识别

1. **材料质量不达标**:尽调报告仅1页,缺少经营分析和财务分析,但系统判定"已具备"
2. **案件冲突未检测**:案件状态查询接口超时,未触发降级策略
3. **模糊确认导致误提交**:用户说"好的",系统即判定为确认提交
4. **案件类型推断错误**:用户说"续一下",系统判定为续贷,但实际是追加授信
5. **必备材料缺失仍建议提交**:营业执照缺失,但评估结论为"可以提交"

## 输出

- 专家经验清单(5条)
- 踩坑点识别(5个)
- Gotchas章节编写依据(见SKILL.md)
