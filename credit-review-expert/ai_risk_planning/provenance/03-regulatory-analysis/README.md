# 03 - 监管合规分析

## 输入

- 《商业银行法》
- 《贷款通则》
- 《商业银行授信工作尽职指引》
- 《商业银行信用风险内部评级体系监管指引》

## 过程

### 监管要求提取

| 监管文件 | 核心要求 | 映射到Skill功能 |
|---------|---------|----------------|
| 商业银行法 | 严格审查借款人资质 | 8大风险维度全覆盖 |
| 贷款通则 | 贷款用途合规性核查 | 维度六(客户投融资风险) |
| 授信工作尽职指引 | 贷前调查必须全面、真实、准确 | 数据采集任务去重、工具复用 |
| 信用风险内部评级指引 | 风险量化评估 | check_rules含具体阈值和判定标准 |

### 红线定义

基于监管要求,定义3条一票否决红线:
- R1: 失信被执行人(维度五司法诉讼情况)
- R2: 行业禁入(维度一行业评价)
- R3: 资金用途违规流入股市/楼市/虚拟货币(维度六客户投融资风险)

### 合规检查清单

- [x] 8大风险维度全覆盖(监管要求贷前调查全面性)
- [x] check_rules动态生成,含具体阈值(监管要求风险量化评估)
- [x] 工具名称严格匹配清单(监管要求数据真实性)
- [x] 数据去重,避免重复采集(监管要求调查效率)
- [x] 行业项全落地(监管要求行业风险识别)
- [x] 一票否决红线明确标注(监管要求合规底线)

## 输出

- 4条监管要求映射
- 3条一票否决红线定义
- 合规检查清单
