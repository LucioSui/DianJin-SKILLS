# 02 - 专家访谈与经验萃取

## 输入

- 6位资深信贷审批官访谈记录
- 风险经理风险识别要求
- 历史信贷案件分析(2024-2025年)

## 过程

### 专家经验萃取

| 专家经验 | 来源 | 应用到Skill |
|---------|------|------------|
| check_rules必须动态生成,不能照抄模板 | 审批退回案例 | 步骤2强制指令 |
| 同一数据源只生成一个采集任务 | 数据采集最佳实践 | 步骤3强制指令 |
| 工具名称必须严格匹配工具清单 | 系统架构规范 | 第六节工具清单 |
| 行业项必须全部落地,不得合并 | 产品管理规范 | 步骤2强制指令 |
| serves_tasks与required_data必须双向对应 | 数据链路规范 | 步骤5校验项 |

### 踩坑点识别

1. **check_rules照抄模板缺乏动态性**: 生成的check_rules都是通用描述,缺少具体阈值和判定标准
   - **解决**: 步骤2增加强制指令"不得照抄通用模板,必须根据本笔贷款特征动态生成"

2. **采集任务重复生成**: 同一数据源生成多个采集任务,违反去重原则
   - **解决**: 步骤3增加强制指令"同一数据源只生成一个采集任务"

3. **工具名称不匹配工具清单**: 使用自定义工具名称,不在工具清单中
   - **解决**: 步骤3增加强制指令"必须从工具清单中选择工具,不得创建新工具名称"

4. **行业项任务合并或省略**: 产品规则中的多条规则被合并为一个任务
   - **解决**: 步骤2增加强制指令"每条规则生成独立任务,不得合并或省略"

5. **serves_tasks与required_data不对应**: 分析任务的数据需求在采集任务中找不到对应服务关系
   - **解决**: 步骤5增加校验项"核实双向对应"

### 风险特征识别规则

基于专家经验,总结常见风险特征识别规则:
- **新设企业**(成立时间<3年): 经营持续性风险,需增加经营持续性分析针对项
- **大额授信**(超行业平均2倍): 资金用途专项核查,过度融资合理性分析
- **科技企业**: 知识产权风险,研发投入合理性,退出路径可行性
- **贸易企业**: 存货周转率,应收账期,资金闭环路径追踪
- **制造业**: 水电用量与产量交叉验证,固定资产折旧政策激进性评估
