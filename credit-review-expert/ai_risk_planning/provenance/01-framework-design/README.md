# 01 - 框架设计与架构定义

## 输入

- 原SKILL.md(466行,6步Workflow)
- 信贷风控任务规划需求
- 8大风险筛查维度框架
- 监管文件(《商业银行法》《贷款通则》等)

## 过程

### 需求提取

| 需求项 | 来源 | 应用到Skill |
|--------|------|------------|
| 自顶向下规划逻辑 | 信贷审批流程规范 | 步骤1-5执行流程 |
| 8大风险维度全覆盖 | 监管要求+内部制度 | 步骤2强制指令 |
| 行业项全落地 | 产品管理规范 | 步骤2强制指令 |
| 数据去重 | 数据采集最佳实践 | 步骤3强制指令 |
| 工具复用 | 系统架构设计 | 第六节工具清单 |

### 三层任务体系设计

基于信贷审批规范,定义三层任务体系:
1. 风险分析任务(三类来源:固定项/行业项/针对项)
2. 数据采集任务(由分析任务数据需求驱动)
3. 审批报告任务(汇总分析结论)

### 交互模式判定

**判定结果**: 模式 A - 报告生成型

**判定依据**:
- 输入明确(贷款申请信息+可选文件),输出为完整JSON任务规划清单
- 单次调用即可生成完整输出
- 无多轮对话或迭代精炼需求

### 管线类型判定

**判定结果**: 混合型(数据驱动型为主 + 制度合规型为辅 + 知识萃取型)

**判定依据**:
- **数据驱动型特征**(主): 核心逻辑基于数据分析(8大风险维度的量化检查规则、DSCR测算、财务指标验证、流水匹配度分析)
- **制度合规型特征**(辅): 核心逻辑基于监管文件,强调红线定义、一票否决条件、合规检查
- **知识萃取型特征**(弱): 风险特征识别、动态审查深度决策依赖专家经验

- 交互模式判定报告: 模式A
