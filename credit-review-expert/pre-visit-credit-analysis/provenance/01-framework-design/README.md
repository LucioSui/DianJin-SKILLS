# 步骤1:框架设计与业务逻辑定义

## 输入
- 原pre-visit-credit-analysis SKILL.md(213行)
- 《商业银行贷款业务管理办法》(银监会令2010年第2号)
- 《关于加强贷款风险管理的通知》(银监发〔2020〕22号)
- 银行内部贷前调查操作规程

## 过程
1. 提取原SKILL.md中的核心业务规则:
   - 五大数据维度收集(工商/司法/舆情/行内/征信)
   - 红旗/黄旗信号分级机制
   - 一票否决条件V1-V6定义
   - 访前一页纸输出模板
2. 设计材料三级分类:
   - 必备材料:工商登记信息、司法判决文书、征信报告(如有授权)
   - 建议材料:新闻舆情记录、行内存贷数据
   - 补充材料:企业荣誉资质、对外投资企业记录
3. 提取8条核心业务规则:
   - 规则1:股权穿透至最终自然人,至少3层
   - 规则2:注册资本实缴比例<30%标注黄旗
   - 规则3:失信被执行人触发V1一票否决
   - 规则4:近2年行政处罚≥3次标注红旗
   - 规则5:授信占用率>90%标注黄旗
   - 规则6:征信查询次数>6次/6个月标注黄旗
   - 规则7:对外担保总额>净资产50%标注黄旗
   - 规则8:一票否决条件V1-V6优先检查,触发即标注
4. 定义降级策略:
   - 工商数据接口不可用→网页查询降级
   - 行内CRM不可用→跳过并记录
   - 征信未取得授权→跳过并记录
   - 舆情工具不可用→通用搜索替代

## 输出
- [核心业务规则保护清单](../04-skill-synthesis/README.md)
- [降级策略设计](../../SKILL.md#数据接入-data-sources)
- 8条核心业务规则提取完成,列入保护清单
