# 步骤2:数据集成与管线设计

## 输入
- 核心业务规则提取清单(来自[步骤1](../01-framework-design/README.md))
- 管线类型判定报告:混合型(数据驱动型为主 + 制度合规型为辅)

## 过程
1. 设计五大数据维度集成管线:
   - 工商数据:企查查/天眼查/国家企业信用信息公示系统→API接口
   - 司法数据:凭安征信/企查查司法模块/法院公告网→API接口
   - 舆情数据:百炼Web搜索/新闻舆情系统→Web搜索
   - 行内数据:CRM系统/核心银行系统→内部API
   - 征信数据:人民银行征信系统→内部API(需授权)
2. 设计数据收集顺序:
   - 步骤1:工商基本信息(基础数据,优先收集)
   - 步骤2:司法风险筛查(高风险信号,优先识别)
   - 步骤3:经营舆情扫描(中风险信号,补充分析)
   - 步骤4:行内存贷数据(如有权限,丰富分析维度)
   - 步骤5:外部征信(需授权,如未取得则跳过)
3. 设计降级策略:
   - 每个数据源定义失败处理路径
   - 单一数据源失败不中断整体流程
   - 失败原因显式记录,不在报告中淡化
4. 设计数据脱敏规则:
   - 客户名称/统一信用代码使用占位符
   - 征信报告中的身份证号/银行账号替换为[证件号码]/[银行账号]
   - 行内金额数据可使用区间描述
