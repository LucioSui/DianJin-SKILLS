# 步骤2:规则提取与业务逻辑定义

## 输入
- 监管要求映射表(来自[步骤1](../01-regulatory-review/README.md))
- 原credit-collateral-risk-mgmt SKILL.md(288行)
- 银行内部押品管理制度(如有)

## 过程
1. 提取原SKILL.md中的核心业务规则:
   - 押品分类与抵押率上限(9类押品)
   - 评估要求清单(6项检查)
   - 登记要求矩阵(7类押品登记机关)
   - 监控频率与预警阈值(6个监控维度)
   - 处置流程(4步处置程序)
2. 设计材料三级分类:
   - 必备材料:权属证明、评估报告、登记证明、保险单(房产/设备/存货)
   - 建议材料:历史评估报告、租赁合同、规划许可证
   - 补充材料:现场照片、市场询价记录、专家意见
3. 提取8条核心业务规则:
   - 规则1:押品合法所有,无司法限制
   - 规则2:评估报告在有效期内
   - 规则3:登记手续完备有效
   - 规则4:抵押率不超过品类上限
   - 规则5:存续期监控按频率执行
   - 规则6:一票否决条件C1-C6优先检查
   - 规则7:价值补足机制15工作日时限
   - 规则8:处置程序合法合规,拍卖底价≥评估价70%

## 输出
- 抵押率上限表(9类押品)
- 评估要求清单(6项)
- 登记要求矩阵(7类)
- 监控频率与预警阈值表(6维度)
- 一票否决条件定义(C1-C6)
