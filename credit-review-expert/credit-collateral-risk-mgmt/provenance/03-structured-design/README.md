# 步骤3:流程设计与门控机制

## 输入
- 核心业务规则提取清单(来自[步骤2](../02-rule-extraction/README.md))
- 交互模式判定报告:模式B - 步骤门控型

## 过程
1. 设计6步Workflow(增加步骤0先读后写):
   - 步骤0:数据确认与验证(先读后写)
   - 步骤1:押品准入合规性筛查
   - 步骤2:价值评估合理性审查
   - 步骤3:登记手续完备性核验
   - 步骤4:存续期动态监控
   - 步骤5:处置合规性检查
2. 设计门控路径:
   - 步骤1门控:C3触发→立即停止;权属瑕疵→标记风险点继续
   - 步骤2门控:C4触发→红色警示;评估机构非认可/超期→退回
   - 步骤3门控:C2/C5触发→红色警示;轻微瑕疵→标记风险点继续
   - 步骤4门控:黄色预警→生成通知;红色预警→启动补足/压缩敞口
   - 步骤5门控:处置程序违法→纠正后继续
3. 设计先读后写机制(步骤0):
   - 强制运行validate_collateral.py验证数据格式
   - 验证失败返回缺失清单,不得进入后续步骤
4. 设计防模型偷懒指令:
   - 步骤0:不得跳过必填字段检查
   - 步骤1:不得跳过权属清晰度检查
   - 步骤2:不得采纳未经评估或超期失效的押品价值
   - 步骤3:不得跳过优先受偿权核查
   - 步骤4:不得跳过实地查看步骤
   - 步骤5:不得跳过诉讼时效管理

## 输出
- 6步Workflow设计(见[SKILL.md](../../SKILL.md))
- 门控路径设计(通过/不通过/预警)
- 先读后写机制(步骤0)
- 防模型偷懒指令(6条)

## Workflow设计摘要
| 步骤 | 执行内容 | 门控条件 | 分支路径 |
|------|---------|---------|---------|
| 步骤0 | 数据确认与验证 | 数据完整性 | ✅完整→步骤1 / ❌缺失→返回清单 |
| 步骤1 | 准入合规性筛查 | C3触发/权属瑕疵 | ✅通过→步骤2 / ❌C3→停止 / ⚠️瑕疵→高亮继续 |
| 步骤2 | 价值评估合理性审查 | C4触发/评估无效 | ✅通过→步骤3 / ❌C4/无效→退回 / ⚠️偏差20-30%→高亮继续 |
| 步骤3 | 登记手续完备性核验 | C2/C5触发/轻微瑕疵 | ✅通过→步骤4 / ❌C2/C5→停止 / ⚠️瑕疵→高亮继续 |
| 步骤4 | 存续期动态监控 | 黄色/红色预警 | ✅正常→步骤5 / ⚠️黄色→通知继续 / ❌红色→补足/压缩 |
| 步骤5 | 处置合规性检查 | 程序违法 | ✅合规→输出报告 / ❌违法→纠正继续 |
