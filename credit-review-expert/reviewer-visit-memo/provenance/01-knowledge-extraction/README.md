# 步骤1:知识萃取与业务逻辑定义

## 输入
- 原reviewer-visit-memo SKILL.md(274行)
- 《商业银行贷款业务管理办法》(银监会令2010年第2号)
- 《商业银行授信工作尽职指引》(银监发〔2004〕51号)
- 《商业银行信用风险内部评级体系监管指引》
- 银行内部审批委员会议事规则

## 过程
1. 提取原SKILL.md中的核心业务规则:
   - 六维度走访纪要框架(现场观感/经营判断/人物印象/资金流向/风险信号/与CM沟通结论)
   - 一票否决条件S1-S6定义
   - 增量更新与信息修正机制
   - 事实优先原则(每条记录须注明来源)
2. 设计材料三级分类:
   - 必备材料:访前一页纸、CM信贷经理调查报告、客户基本信息
   - 建议材料:走访邀请记录、历史走访纪要
   - 补充材料:审批专员口述观察内容
3. 提取8条核心业务规则:
   - 规则1:事实优先,每条记录须有观察来源
   - 规则2:CM交叉验证,偏差须显式标注
   - 规则3:信息修正可追溯,保留原记录
   - 规则4:敏感信号必报(停工/欠薪/民间融资)
   - 规则5:笔记归档确认
   - 规则6:一票否决条件S1-S6优先检查
   - 规则7:量化优先,不得用模糊表述
   - 规则8:版本可追溯,增量更新保留历史

## 输出
- 核心业务规则提取清单(8条规则)
- 监管要求映射表(4部监管文件→核心要求)
- 材料三级分类设计

## 关键决策
- 管线类型判定:知识萃取型(主) + 制度合规型(辅)
  - 知识萃取型特征:核心逻辑来源于审批专员实地走访经验、专家观察技巧、现场判断策略
  - 制度合规型特征:核心逻辑来源于监管文件,强调一票否决红线(S1-S6)、审批独立性要求
