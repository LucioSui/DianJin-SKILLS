# 步骤2:专家访谈与经验萃取

## 输入
- 核心业务规则提取清单(来自[步骤1](../01-knowledge-extraction/README.md))
- 审批专员实地走访经验(专家访谈)
- 银行内部走访纪要模板与最佳实践

## 过程
1. 萃取审批专员实地走访专家经验:
   - 现场观感:如何估算开工率、识别经营场所真实性、发现环保/安全隐患
   - 经营判断:如何核查合同原件、评估库存合理性、交叉验证行业景气度
   - 人物印象:如何识别回避与引导迹象、判断配合度是否异常
   - 资金流向:如何发现体外循环迹象、识别关联交易、核实资金用途一致性
   - 风险信号:如何识别显性风险(停工/欠薪/查封)与隐性风险(信息回避/数据矛盾)
2. 设计六维度详细框架:
   - 每个维度包含检查项、记录要点、偏差标记/交叉验证/风险信号/风险阈值
   - 增加执行要点,明确量化标准与注意事项
3. 提取4条专家经验规则:
   - 经验1:开工率须估算具体比例,不得写"基本正常"等模糊表述
   - 经验2:合同核查优先看原件或盖章复印件,口头声称须标注"未核验原件"
   - 经验3:"配合度"非越高越好,过度配合可能是事先编排的信号
   - 经验4:资金流核实优先看银行流水原件或网银截图,口头描述须标注来源

## 输出
- 六维度详细框架(检查项/记录要点/偏差标记/交叉验证/风险信号/执行要点)
- 专家经验规则提取清单(4条)
- 详细框架文档: `references/visit-memo-dimensions.md`
