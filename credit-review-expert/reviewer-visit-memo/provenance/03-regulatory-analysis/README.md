# 步骤3:监管分析与合规设计

## 输入
- 核心业务规则提取清单(来自[步骤1](../01-knowledge-extraction/README.md))
- 交互模式判定报告:模式D - 对话辅助型(Conversational Assistant)

## 过程
1. 设计8步Workflow(增加步骤0先读后写+步骤7一票否决自检):
   - 步骤0:数据确认与验证(先读后写)
   - 步骤1:首次输入处理(完整纪要生成)
   - 步骤2:增量更新处理
   - 步骤3:信息修正处理
   - 步骤4:针对性追问
   - 步骤5:不保存模式处理
   - 步骤6:自然退出
   - 步骤7:一票否决自检(门控步骤)
2. 设计对话辅助型交互模式:
   - 开始阶段:确认客户信息、检查访前一页纸、引导口述
   - 中间阶段:实时结构化整理、针对性追问、支持增量更新与信息修正
   - 结束阶段:一票否决自检、生成纪要、确认保存、提供审批关注点
3. 设计上下文管理机制:
   - 上下文累积规则:多轮对话持续累积六维度信息
   - 上下文清理规则:用户说"重新开始"时清空,超过24小时未更新提示确认
4. 设计一票否决条件S1-S6:
   - S1:经营场所虚假或已被查封,与申报严重不符
   - S2:发现企业已实质性停工超过30天,且无合理恢复计划
   - S3:资金实际用途与申请用途严重不符(如流贷用于购房/炒股)
   - S4:发现明显民间融资/非法集资迹象,且金额较大
   - S5:实控人失联或拒绝见面,且无法提供合理解释
   - S6:现场发现重大环保/安全隐患,已被政府部门责令整改

## 输出
- 8步Workflow设计(含分支路径)
- 交互模式设计(模式D - 对话辅助型)
- 上下文管理机制设计
- 一票否决条件S1-S6定义

## 关键决策
- 交互模式判定为模式D(对话辅助型),需增加Interaction Pattern和Context Management章节
- Workflow设计支持多轮对话、增量更新、信息修正、不保存模式、自然退出
- 步骤7强制一票否决自检,触发须红色标注并建议暂停审批
