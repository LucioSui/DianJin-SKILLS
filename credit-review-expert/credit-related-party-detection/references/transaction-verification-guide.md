# 四流合一验证指南

> **数据有效期至**: 2027-12-31  
> **关联监管文件**: 《商业银行贷款业务管理办法》(银监会令2010年第2号)、《关于加强贷款风险管理的通知》(银监发〔2020〕22号)

---

## 一、四流合一验证概述

四流合一验证是关联交易真实性核查的核心方法,通过**合同流、发票流、物流/服务流、资金流**四个维度的交叉验证,识别虚构交易、资金空转、转移定价等异常特征。

### 1.1 四流定义

| 流程 | 定义 | 关键凭证 | 验证要点 |
|------|------|---------|---------|
| **合同流** | 交易双方签署的商务合同 | 采购合同/销售合同/服务协议 | 合同条款是否公允、是否与实际交易一致 |
| **发票流** | 交易开具的增值税发票 | 增值税专用发票/普通发票 | 发票金额/税率/品名是否与合同一致 |
| **物流/服务流** | 货物交付或服务提供的实际过程 | 出库单/入库单/物流单据/服务验收单 | 是否有真实的货物移动或服务交付 |
| **资金流** | 交易资金的支付和收取 | 银行流水/支付凭证 | 资金流向是否与合同/发票一致,是否存在回流 |

### 1.2 验证原则

- **四流一致**: 合同、发票、物流、资金流在金额、时间、对象上相互匹配
- **两流匹配不足**: 仅有合同+发票但无物流/资金流对应,标记"疑似虚构交易"
- **异常必查**: 任一流程存在异常,必须深入核查其他流程

## 二、四流合一验证流程

### 2.1 第一步:合同流验证

**验证内容**:
1. 合同主体是否为关联方(对照关联方清单)
2. 合同金额是否合理(与非关联交易价格对比)
3. 合同条款是否公允(付款条件、交货期限、违约责任)
4. 合同签署时间是否合理(是否存在倒签)

**异常信号**:
- 🔴 合同主体为未披露的关联方
- 🔴 合同价格偏离市场价 ≥ 30%
- 🟡 合同价格偏离市场价 20%-30%
- 🟡 合同签署时间与物流/资金流时间不匹配(如合同签署晚于交货)

**数据来源**:
- 企业提供的交易合同
- 企查查/天眼查(合同主体背景调查)
- 行业价格基准(市场价对比)

### 2.2 第二步:发票流验证

**验证内容**:
1. 发票开具方/接收方是否与合同主体一致
2. 发票金额是否与合同金额一致(允许合理尾差)
3. 发票品名是否与合同标的、实际物流一致
4. 发票税率是否符合税法规定

**异常信号**:
- 🔴 发票开具方/接收方与合同主体不一致(存在第三方代开)
- 🔴 发票金额与合同金额差异 > 10%
- 🟡 发票品名模糊(如"咨询服务费"无具体内容)
- 🟡 发票开具时间异常(如集中开票、跨期开票)

**数据来源**:
- 企业提供的增值税发票
- 税务系统(发票真伪验证)

### 2.3 第三步:物流/服务流验证

**验证内容**:
1. 是否有真实的货物移动(出库单、入库单、物流单据)
2. 物流方向是否与合同一致(如采购合同应有货物入库)
3. 物流时间是否与合同/发票时间匹配
4. 服务类交易是否有服务验收单/成果报告

**异常信号**:
- 🔴 无物流单据但有大额交易(疑似虚构交易)
- 🔴 物流方向与合同相反(如销售合同但无出库单)
- 🟡 物流时间与合同/发票时间差异 > 30天
- 🟡 服务类交易无验收单或验收标准模糊

**数据来源**:
- 企业提供的出库单、入库单、物流单据
- 物流公司查询(物流轨迹)
- 实地走访(仓储现场查看)

### 2.4 第四步:资金流验证

**验证内容**:
1. 资金支付方/接收方是否与合同主体一致
2. 资金金额是否与合同/发票金额一致
3. 资金流向是否存在回流(如A支付给B,B又转回给A)
4. 资金支付时间是否与合同/物流时间匹配

**异常信号**:
- 🔴 资金支付方/接收方与合同主体不一致
- 🔴 资金回流(A→B→A或A→B→C→A)
- 🔴 资金金额与合同/发票金额差异 > 10%
- 🟡 资金支付时间异常(如提前支付、延迟支付)

**数据来源**:
- 企业银行流水(需客户授权)
- 征信系统(贷款资金流向)
- 实地走访(财务人员访谈)

## 三、四流合一交叉验证矩阵

### 3.1 标准验证矩阵

| 验证维度 | 合同流 | 发票流 | 物流/服务流 | 资金流 | 验证结论 |
|---------|--------|--------|------------|--------|---------|
| **主体一致性** | 合同甲方/乙方 | 发票开具方/接收方 | 发货方/收货方 | 支付方/接收方 | 四者一致 → ✅ 通过 |
| **金额一致性** | 合同金额 | 发票金额 | 货物价值/服务价值 | 支付金额 | 差异 < 5% → ✅ 通过 |
| **时间一致性** | 合同签署时间 | 发票开具时间 | 交货/服务时间 | 资金支付时间 | 时间逻辑合理 → ✅ 通过 |
| **标的真实性** | 合同标的 | 发票品名 | 实际货物/服务 | 资金用途 | 四者匹配 → ✅ 通过 |

### 3.2 异常判定规则

| 匹配情况 | 匹配流程数 | 判定结论 | 处理措施 |
|---------|-----------|---------|---------|
| 四流完全匹配 | 4流 | ✅ 交易真实 | 正常记录 |
| 三流匹配,一流缺失 | 3流 | ⚠️ 需补充验证 | 要求客户提供缺失流程凭证 |
| 两流匹配(合同+发票) | 2流 | 🚩 疑似虚构交易 | 标记红旗,深度核查物流/资金流 |
| 两流匹配(合同+资金) | 2流 | 🚩 疑似虚假发票 | 标记红旗,核查发票真伪 |
| 一流匹配 | 1流 | 🚩 高度可疑 | 标记红旗,暂停授信,启动专项调查 |
| 四流均不匹配 | 0流 | 🚩 明确虚构 | 触发R1一票否决,上报合规部门 |

## 四、典型异常场景识别

### 4.1 资金空转

**特征**:
- 关联企业间频繁大额转账(月均超过3笔或单月超500万)
- 无真实贸易背景(无合同/发票/物流对应)
- 资金流向形成闭环(A→B→C→A)

**四流验证结果**:
- 合同流: 无合同或合同标的模糊
- 发票流: 无发票或发票品名模糊
- 物流/服务流: 无物流单据
- 资金流: 频繁大额转账,存在回流

**处理措施**: 触发R1一票否决,立即上报合规部门

### 4.2 转移定价

**特征**:
- 关联交易价格偏离市场价 ≥ 20%
- 采购价偏高或销售价偏低(利润转移至关联方)
- 无法提供合理商业解释

**四流验证结果**:
- 合同流: 合同价格异常
- 发票流: 发票金额与合同一致但偏离市场价
- 物流/服务流: 物流真实但货物价值与价格不匹配
- 资金流: 资金支付金额异常

**处理措施**: 触发R2一票否决(偏离≥30%)或标注黄旗(偏离20%-30%)

### 4.3 虚构交易

**特征**:
- 仅有合同/发票,无物流/资金流对应
- 物流与交易方向不符(如销售合同但无出库单)
- 资金支付方/接收方与合同主体不一致

**四流验证结果**:
- 合同流: 合同存在但条款模糊
- 发票流: 发票存在但品名模糊
- 物流/服务流: 无物流单据或物流方向不符
- 资金流: 无资金支付或支付方不一致

**处理措施**: 触发R1一票否决,立即上报合规部门

### 4.4 利润转移

**特征**:
- 借款企业毛利率/净利率显著低于非关联方交易
- 利润主要流向关联方
- 关联交易占比 > 50%

**四流验证结果**:
- 合同流: 合同价格偏低(销售)或偏高(采购)
- 发票流: 发票金额与合同一致
- 物流/服务流: 物流真实
- 资金流: 资金流向关联方

**处理措施**: 触发R3一票否决(系统性转移)或标注黄旗(单笔异常)

## 五、四流合一验证工具

### 5.1 验证脚本

```python
# 四流合一验证伪代码
def verify_four_flows(contract, invoice, logistics, fund_flow):
    """
    四流合一验证
    
    Args:
        contract: 合同信息(主体、金额、时间、标的)
        invoice: 发票信息(开具方、金额、时间、品名)
        logistics: 物流信息(发货方、收货方、时间、货物)
        fund_flow: 资金流信息(支付方、接收方、金额、时间)
    
    Returns:
        dict: 验证结果(匹配流程数、判定结论、异常信号)
    """
    match_count = 0
    
    # 1. 主体一致性验证
    if verify_subject_consistency(contract, invoice, logistics, fund_flow):
        match_count += 1
    
    # 2. 金额一致性验证
    if verify_amount_consistency(contract, invoice, fund_flow):
        match_count += 1
    
    # 3. 时间一致性验证
    if verify_time_consistency(contract, invoice, logistics, fund_flow):
        match_count += 1
    
    # 4. 标的真实性验证
    if verify_subject_matter(contract, invoice, logistics, fund_flow):
        match_count += 1
    
    # 判定结论
    if match_count == 4:
        return {"match_count": 4, "conclusion": "交易真实", "signal": "✅"}
    elif match_count == 3:
        return {"match_count": 3, "conclusion": "需补充验证", "signal": "⚠️"}
    elif match_count == 2:
        return {"match_count": 2, "conclusion": "疑似虚构交易", "signal": "🚩"}
    else:
        return {"match_count": match_count, "conclusion": "高度可疑", "signal": "🚩"}
```

### 5.2 验证清单

| 验证项 | 验证方法 | 通过标准 | 不通过处理 |
|--------|---------|---------|-----------|
| 主体一致性 | 比对合同/发票/物流/资金流主体 | 四者一致 | 标注异常,深度核查 |
| 金额一致性 | 比对合同/发票/资金流金额 | 差异 < 5% | 差异5%-10%标注黄旗,>10%标注红旗 |
| 时间一致性 | 检查合同/发票/物流/资金流时间逻辑 | 时间逻辑合理 | 时间倒挂标注红旗 |
| 标的真实性 | 比对合同标的/发票品名/实际货物 | 四者匹配 | 不匹配标注红旗 |
| 资金回流检测 | 追踪资金流向 | 无回流 | 发现回流触发R1 |

## 六、数据缺失降级策略

### 6.1 合同流缺失

- **降级方案**: 使用发票+物流+资金流三流验证
- **风险提示**: "合同流缺失,交易真实性验证可能不完整"
- **处理措施**: 要求客户补充提供合同,无法提供时标注"疑似虚构交易"

### 6.2 发票流缺失

- **降级方案**: 使用合同+物流+资金流三流验证
- **风险提示**: "发票流缺失,无法验证税务合规性"
- **处理措施**: 要求客户补充提供发票,无法提供时标注"税务合规风险"

### 6.3 物流/服务流缺失

- **降级方案**: 使用合同+发票+资金流三流验证
- **风险提示**: "物流/服务流缺失,疑似虚构交易"
- **处理措施**: 标记红旗,深度核查,无法核实触发R1

### 6.4 资金流缺失

- **降级方案**: 使用合同+发票+物流三流验证
- **风险提示**: "资金流缺失,无法验证资金用途"
- **处理措施**: 要求客户提供银行流水,无法提供时标注"资金用途不明"

## 七、常见误区

### #1: 仅凭合同+发票就判定交易真实
- **错误做法**: 合同和发票匹配就认定交易真实,未检查物流和资金流
- **正确做法**: 必须执行四流合一验证,仅有两流匹配标记"疑似虚构交易"

### #2: 忽略资金回流检测
- **错误做法**: 仅检查资金支付,未追踪资金流向
- **正确做法**: 必须追踪资金流向,发现回流(A→B→A)触发R1

### #3: 时间逻辑验证不严格
- **错误做法**: 合同签署时间晚于交货时间未标注异常
- **正确做法**: 时间倒挂(合同晚于交货/发票晚于付款)必须标注红旗

### #4: 金额差异容忍度过高
- **错误做法**: 合同金额与发票金额差异15%仍判定通过
- **正确做法**: 差异 < 5%通过,5%-10%标注黄旗,>10%标注红旗
