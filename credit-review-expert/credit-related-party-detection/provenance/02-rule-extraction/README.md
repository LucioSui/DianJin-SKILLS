# 步骤2:规则提取与业务逻辑定义

## 输入
- 监管要求映射表(来自[步骤1](../01-regulatory-review/README.md))
- 原credit-related-party-detection SKILL.md(270行)
- 银行内部关联交易管理办法(如有)

## 过程
1. 提取原SKILL.md中的核心业务规则:
   - 六维关联方识别(股权/管理层/家族/交易/担保/隐性)
   - 关联交易异常特征识别(7类异常特征)
   - 关联风险评估(授信集中度/经营独立性/担保链/财务健康)
   - 风险分级与处置建议(高/中/低三级)
2. 设计材料三级分类:
   - 必备材料:股权穿透数据、征信报告、企业财务报表、担保合同
   - 建议材料:历史关联交易报告、实地走访记录、客户关联方声明
   - 补充材料:同地址/同电话/同财务人员等隐性关联信号
3. 提取8条核心业务规则:
   - 规则1:股权穿透至最终自然人,至少3层
   - 规则2:六维关联全部排查,不得选择性忽略
   - 规则3:四流合一验证(合同/发票/物流/资金)
   - 规则4:关联交易占比=关联交易金额/同类交易总金额
   - 规则5:担保圈≥3家形成担保圈,≥5家大型担保圈
   - 规则6:一票否决条件R1-R6优先检查,触发即标注
   - 规则7:隐性关联六项信号检查(同地址/同电话/同邮箱/同财务人员/同审计机构/同步变更)
   - 规则8:客户拒绝配合调查须显式记录并给出审慎建议

## 输出

## 关键决策
- 保留原有六维关联方识别框架,不做删减
- 保留原有7类异常特征识别,增加四流合一验证要求
- 保留原有R1-R6一票否决条件,不做修改
- 增加步骤0(先读后写)强制数据验证
- 增加门控路径标注(模式B交互模式)
