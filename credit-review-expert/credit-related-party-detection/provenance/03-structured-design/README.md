# 步骤3:流程设计与门控机制

## 输入
- 核心业务规则提取清单(来自[步骤2](../02-rule-extraction/README.md))
- 交互模式判定报告:模式B - 步骤门控型

## 过程
1. 设计5步Workflow(增加步骤0先读后写):
   - 步骤0:数据确认与验证(先读后写)
   - 步骤1:关联方识别(六维全覆盖)
   - 步骤2:关联交易异常特征识别(四流合一验证)
   - 步骤3:关联风险评估(集中度+独立性+担保链+财务健康)
   - 步骤4:报告输出与处置建议
2. 设计门控路径:
   - 步骤0门控:客户名称/统一信用代码为空→立即停止
   - 步骤1门控:六维关联全部识别完成→进入步骤2;部分维度数据缺失→标注"数据缺失",继续
   - 步骤2门控:触发R1-R6→红色警示,可进入步骤3但须在报告顶部标注
   - 步骤3门控:风险评估完成,无新增一票否决触发→进入步骤4
3. 设计先读后写机制(步骤0):
   - 强制运行validate_related_party.py验证数据完整性
   - 验证失败不得进入后续流程
4. 设计防模型偷懒指令:
   - 步骤1:不得跳过任何维度,六类关联关系须全部排查
   - 步骤1:股权穿透必须穿透至最终自然人,不得停留在中间层法人
   - 步骤2:不得仅凭"合同+发票"两流匹配就判定交易真实,必须检查物流/资金流
   - 步骤2:发现资金空转或虚构交易嫌疑须立即标注,不得忽略
   - 步骤3:不得将"关联交易占比<30%"直接判定为低风险,必须综合四流合一验证结果
   - 步骤4:不得遗漏任何关联方,必须全部列示

## 关键设计
- 步骤1(关联方识别):六维关联并行执行,全部完成后汇总
- 步骤2(异常特征识别):7类异常特征串行检查,R1-R6一票否决优先
- 步骤3(风险评估):四维度评估(集中度/独立性/担保链/财务健康)并行执行
- 步骤4(报告输出):汇总前四步结果,生成结构化报告
