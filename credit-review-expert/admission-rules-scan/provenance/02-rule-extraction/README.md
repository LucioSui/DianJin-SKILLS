# 02 - 规则提取与结构化

## 输入

- 原SKILL.md(214行,4步执行流程)
- 监管要求映射表(来自01-regulatory-review)
- 信贷审批官准入评估经验

## 过程

### 规则三级分类设计

基于监管要求和产品管理办法,定义三级规则体系:

| 规则级别 | 定义 | 典型规则数 | 应用场景 |
|---------|------|-----------|---------|
| P0固定项 | 全量客户必检 | 8-12条 | 经营年限、征信无逾期、注册资本 |
| P1行业项 | 行业专属限制 | 3-6条 | 房地产、两高一剩、鼓励类行业 |
| P2针对项 | 个案补充规则 | 2-4条 | 大额授信、关联交易、担保方式 |

### 三态评估机制设计

| 结果 | 判定标准 | 输出要求 |
|------|---------|---------|
| ✅ 通过 | 客户条件完全满足规则阈值 | 列出规则名及匹配值 |
| ❌ 不满足 | 客户条件不满足规则阈值 | 列出规则名、阈值要求、实际值、差距 |
| ⏸️ 需确认 | 关键信息缺失,无法判断 | 列出缺失字段及建议补充材料 |

### 一票否决前置机制

**设计依据**: 监管要求明确,某些条件一旦触发,客户即不具备贷款资格,无需继续评估其他规则。

**执行流程**:
1. 加载客户画像和产品规则
2. **先执行A1-A6一票否决扫描**
3. 如触发任一条件 → 立即终止,输出"准入不通过"
4. 如未触发 → 继续逐条规则匹配

### 差距量化算法

```
差距绝对值 = |阈值要求 - 客户实际值|
差距相对比例 = 差距绝对值 / 阈值要求 × 100%
```

**示例**:
- 要求:资产负债率≤70%
- 实际:75%
- 差距绝对值:5%
- 差距相对比例:5%/70% = 7.14%

## 输出

- 规则三级分类体系 → 应用于步骤2(加载产品准入规则)
- 三态评估机制 → 应用于步骤4(逐条规则匹配)
- 一票否决前置机制 → 应用于步骤3
- 差距量化算法 → 应用于质量要求第4条
