# 04 - Skill合成与迭代

## 输入

- 原SKILL.md(214行)
- 监管要求映射表(来自01-regulatory-review)
- 规则提取结果(来自02-rule-extraction)
- 流程设计方案(来自03-structured-design)

## 过程

### 核心业务逻辑保护清单

改造过程中严格保护以下8条核心业务规则,未做任何修改:

1. **规则三级分类**:P0固定项(全量必检)/P1行业项(行业专属)/P2针对项(个案补充)
2. **一票否决前置**:A1-A6条件触发即终止,直接输出"准入不通过"
3. **三态评估机制**:符合(✅)/不满足(❌)/需确认(⏸️)
4. **阈值严格对照**:数值类规则精确到小数点后两位
5. **差距量化**:不符合项给出具体差距数值(绝对值+相对比例)
6. **信息不足不推断**:缺少数据时标记"需确认",不得推断为通过或不满足
7. **6条一票否决红线**:A1(未结清逾期)/A2(淘汰类行业)/A3(失信被执行人)/A4(严重失信行为)/A5(过度担保)/A6(洗钱黑名单)
8. **5种典型应用场景**:新客户首次申请/产品升级/跨产品线/规则库更新/方案设计预检

### 改造前后对比

| 维度 | 改造前 | 改造后 | 说明 |
|------|--------|--------|------|
| SKILL.md行数 | 214行 | ~400行 | 增加provenance体系、测试用例、审计日志 |
| Frontmatter | 2字段(name/description) | 10字段 | 增加target_role/business_domain/version等 |
| Workflow | 4步(无步骤0) | 6步(步骤0-5) | 增加先读后写、门控机制 |
| Constraints | 5条 | 7条 | 增加禁止跳过步骤、红线执行强制 |
| Output Format | 代码块形式 | 6章节表格 | 结构化输出,下游可解析 |
| provenance/ | 无 | 4个子目录 | 制度合规型管线 |
| test_cases.yaml | 无 | 待创建 | 覆盖4种测试类型 |

### 关键创新点

1. **步骤0先读后写机制**: 强制数据验证,防止数据质量问题导致后续流程失败
2. **一票否决门控**: 步骤3增加✅/❌分支路径,确保流程可控
3. **防偷懒指令**: 步骤0/3/4增加"不得跳过"、"必须精确比对"等对抗指令
4. **结构化输出**: 6章节表格+下游可解析字段,确保credit-due-diligence Skill可靠提取
5. **制度合规型管线溯源**: 监管文件(4部)+规则提取(8条)+流程设计(6步),真实反映生产过程
6. **差距量化算法**: 明确绝对值和相对比例计算公式,确保改进建议可操作
