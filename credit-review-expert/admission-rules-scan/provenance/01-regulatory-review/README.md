# 01 - 监管文件研读

## 输入

- 《商业银行贷款业务管理办法》(银监会令2010年第2号)
- 《商业银行信用风险内部评级体系监管指引》
- 《商业银行大额风险暴露管理办法》(银保监会令2018年第1号)
- 银行内部信贷政策与产品管理办法

## 过程

### 监管要求提取

| 监管文件 | 核心要求 | 映射到Skill功能 |
|---------|---------|----------------|
| 商业银行贷款业务管理办法 | 贷款对象与条件明确 | P0固定项规则定义 |
| 信用风险内部评级体系监管指引 | 客户评级与准入标准 | 三态评估(符合/不满足/需确认) |
| 大额风险暴露管理办法 | 集中度管控 | P1行业项规则(集中度限制) |
| 内部信贷政策与产品管理办法 | 具体准入规则来源 | P2针对项规则(个案补充) |

### 一票否决红线定义

基于监管要求,定义6条一票否决红线(A1-A6):
- A1: 未结清逾期(征信M2及以上) - 违反贷款对象条件
- A2: 淘汰类行业 - 违反产业政策要求
- A3: 失信被执行人 - 违反合规要求
- A4: 严重失信行为记录 - 违反诚信原则
- A5: 过度担保/互保联保 - 违反大额风险暴露管理办法
- A6: 洗钱/恐怖融资黑名单 - 违反反洗钱法规

### 合规检查清单

- [x] 规则优先序明确(P0>P1>P2)
- [x] 一票否决前置(A1-A6)
- [x] 阈值严格对照(不得模糊判断)
- [x] 信息不足标记"需确认"(不得推断)
- [x] 差距量化(绝对值+相对比例)

## 输出

- 监管要求映射表 → 应用于步骤2(加载产品准入规则)
- 一票否决红线定义 → 应用于步骤3(一票否决前置扫描)
- 合规检查清单 → 应用于质量要求章节
