# 03 - 流程设计与结构化

## 输入

- 原SKILL.md(214行,4步执行流程)
- 监管要求映射表(来自01-regulatory-review)
- 规则提取结果(来自02-rule-extraction)

## 过程

### 流程重构决策

| 决策项 | 原设计 | 新设计 | 依据 |
|--------|--------|--------|------|
| 步骤数量 | 4步 | 6步(增加步骤0和步骤5) | 先读后写机制,报告生成独立步骤 |
| 门控机制 | 无明确分支 | 步骤3标注✅/❌分支 | 模式A-报告生成型,但一票否决需门控 |
| 防偷懒指令 | 无 | 步骤0/3/4增加强制指令 | 对抗模型偷懒 |
| 降级策略 | 无 | 4种场景 | 覆盖系统不可用、数据缺失 |

### 交互模式判定

**判定结果**: 模式 A - 报告生成型(Report Generation)

**判定依据**:
- 输入明确(客户画像+目标产品),输出为完整结构化报告
- 单次调用即可生成完整结果
- 无多轮对话、无需迭代精炼

### 门控路径设计(步骤3一票否决)

**步骤3(一票否决前置扫描)**:
- ✅ 未触发任何A1-A6 → 进入步骤4(逐条规则匹配)
- ❌ 触发任一A1-A6 → **立即停止**,输出"准入不通过"+进入步骤5(生成扫描报告)

### 输出格式结构化设计

**下游兼容性要求**:
- 输出可被 credit-due-diligence Skill 解析
- 6个章节表格+下游可解析字段
- 关键输出字段标注数据类型(enum/array/int)

**免责声明要求**:
- 引用 shared/disclaimer-template.md 模板
- 报告末尾必须包含"不构成最终审批意见"声明

## 输出

- 6步Workflow设计 → 应用于SKILL.md执行流程章节
- 门控路径设计 → 应用于步骤3
- 输出格式结构化 → 应用于Output Format章节
- 免责声明模板 → 应用于Output Format章节
