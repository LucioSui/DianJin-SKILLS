# 02 - 规则提取与结构化

## 输入

- 原SKILL.md(281行,4步校验流程)
- 监管要求映射表(来自01-regulatory-review)
- 信贷审批官进件检查经验

## 过程

### 材料三级分类设计

基于监管要求和内部操作规程,定义三级材料体系:

| 材料级别 | 定义 | 典型材料数 | 处理规则 |
|---------|------|-----------|---------|
| 必备(阻断级) | 监管或行内制度明确要求,缺失则无法审查 | 6项 | 缺失即阻断,须补充后方可进件 |
| 建议(重要级) | 对风险判断有重要支撑作用,缺失影响审查质量 | 5项 | 缺失标注建议补充,可先行进件但须限期补齐 |
| 补充(参考级) | 有助于全面了解客户,缺失不影响基本审查 | 5项 | 缺失标注信息提示,鼓励补充 |

### 三态评估机制设计

| 结果 | 判定标准 | 输出要求 |
|------|---------|---------|
| ✅ 通过 | 客户条件完全满足规则阈值 | 列出规则名及匹配值 |
| ❌ 不满足 | 客户条件不满足规则阈值 | 列出规则名、阈值要求、实际值、差距 |
| ⏸️ 需确认 | 缺少数据无法判定 | 列出缺失字段及补充指引 |

### 核心业务规则提取

从原SKILL.md提取8条核心业务规则:
1. 阻断级不降级:标注为"阻断"的缺失项,必须补充完整后方可进入审查流程
2. 信息一致性:客户基本信息须与营业执照、身份证件等官方证照逐字比对
3. 授权合规:征信授权书须为原件扫描件,授权范围须覆盖本次授信查询
4. 金额合理性:申请金额须在产品政策范围内,用途须具体明确
5. 时效性:营业执照、身份证等证照须在有效期内
6. 材料分级准确:每项材料须正确归类为必备/建议/补充,不得随意降级
7. 一票否决前置:I1-I6须优先检查,触发即终止
8. 补充指引具体化:每项缺失须给出"需要什么、为什么需要、去哪里获取"三级指引

## 输出

- 材料三级分类标准(必备6项/建议5项/补充5项)
- 三态评估机制(✅通过/❌不满足/⏸️需确认)
- 核心业务规则提取(8条)
