# 03 - 流程设计与结构化

## 输入

- 原SKILL.md(281行,4步执行流程)
- 监管要求映射表(来自01-regulatory-review)
- 规则提取结果(来自02-rule-extraction)

## 过程

### 流程重构决策

| 决策项 | 原设计 | 新设计 | 依据 |
|--------|--------|--------|------|
| 步骤数量 | 4步 | 6步(增加步骤0和步骤4) | 先读后写机制,一票否决独立步骤 |
| 门控机制 | 无明确分支 | 步骤4标注✅/❌分支 | 模式A-报告生成型,但一票否决需门控 |
| 防偷懒指令 | 无 | 步骤0/1/2/3/5增加强制指令 | 对抗模型偷懒 |
| 降级策略 | 无 | 4种场景 | 覆盖系统不可用、数据缺失 |

### 交互模式判定

**判定结果**: 模式 A - 报告生成型(Report Generation)

**判定依据**:
- 输入明确(客户信息+申请材料清单),输出为完整结构化检查报告
- 单次调用即可生成完整结果
- 无多轮对话、无需迭代精炼

### 门控路径设计(步骤4一票否决检查)

- ✅ 无触发 → 进入步骤5(生成报告)
- ❌ 触发I1-I6 → **立即停止**,输出"禁止进件"结论,标注触发条件

### 先读后写机制(步骤0)

1. 确认客户身份、案件编号、目标产品
2. 检查必填字段(客户名称、案件编号、目标产品、申请金额、申请材料清单)
3. 运行验证脚本
4. 验证通过后进入步骤1

## 输出

- 6步Workflow设计(步骤0-5)
- 门控路径设计(步骤4一票否决)
- 先读后写机制(步骤0)
