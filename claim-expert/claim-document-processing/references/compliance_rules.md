## 材料审核合规要点

### 时效要求

本Skill定位于"材料审核"环节，按《保险法》要求：

- 收到理赔申请和材料后，应及时作出核定
- 简单案件：5个工作日内完成审核
- 普通案件：15个工作日内完成审核
- 复杂案件：30日内作出核定（最长60日）

本Skill设计目标：单次全流程审核 < 5分钟（22份材料实测 ~285秒），远低于时效要求。

### 材料真实性认定

本Skill**不直接做出**材料真实性判定，而是：

1. **可以做**：识别异常特征并标记（如金额不匹配、时间线矛盾、身份不一致）
2. **不可以做**：独立判定材料为伪造/变造
3. **应当做**：将检测到的异常点结构化输出，供人工核实

异常发现后的处置流程：

```
AI检测到异常 → 标记风险点(type + severity + description)
                → 如 severity=HIGH → 触发 [ALERT]
                → 人工审核员确认
                → 如确认异常 → 转反欺诈调查岗
```

### 个人信息保护

处理理赔材料中的个人信息时遵循以下原则：

| 原则 | 实施措施 |
|------|---------|
| 最小必要 | 仅提取审核必需的关键字段（姓名/金额/日期/诊断） |
| 目的限定 | 提取信息仅用于完整性/一致性校验，不做他用 |
| 不持久化 | 图片Base64仅在API调用期间存在于内存，不落盘 |
| 报告脱敏 | JSON报告中身份证号仅保留前3后4位 |
| 会话隔离 | 不同案件的处理会话完全隔离 |

### 审计要求

按照金融行业审计规范，本Skill的审计日志应满足：

- **完整性**：记录每步操作的输入、输出、时间戳
- **不可篡改**：日志采用追加方式，不可修改历史记录
- **保留期限**：理赔材料审核记录保留不少于保险期满后5年
- **可追溯**：任何输出结果都能回溯到具体的输入和模型版本

## 反欺诈相关规则

### 本Skill可识别的欺诈特征

| 特征类型 | 检测方法 | 对应检查脚本 |
|---------|---------|-------------|
| 重复提交发票 | 发票号/金额/日期完全一致 | check_completeness.py |
| 身份信息不一致 | 跨图姓名/身份证号比对 | check_consistency.py |
| 时间线逻辑矛盾 | 入院/出院/发票/鉴定时间序列校验 | check_consistency.py |
| 费用金额异常 | 发票总额vs清单总额偏差检测 | check_consistency.py |
| 双模型分析分歧 | 两个独立模型对关键字段的判定不一致 | cross_validate.py |

### 不可由AI独立做出的判定

- 材料为伪造/变造
- 案件存在保险欺诈
- 建议拒赔
- 需要报案/报警

以上判定必须由授权人员做出，AI仅提供异常线索。

## 模型输出置信度标准

| 检查维度 | 高置信度(≥0.9) | 中置信度(0.7-0.9) | 低置信度(<0.7) |
|---------|---------------|-----------------|--------------|
| 分类 | 可自动归档 | 可归档但建议人工抽检 | 需人工确认 |
| 完整性 | 结论可信 | 建议人工复核缺失项 | 不可信，须人工判定 |
| 一致性 | 结论可信 | 标记差异点供人工关注 | 触发[ALERT]，暂停流程 |
| 交叉验证 | 结论可信 | 关注分歧点 | 触发[ALERT]，建议人工全面复核 |

## 定期审计机制

| 审计项 | 频率 | 责任人 | 审计内容 |
|--------|------|--------|---------|
| 分类准确率 | 每月 | AI运营 | 抽检100份已归档材料，验证分类正确率 |
| 异常检出率 | 每月 | 风控 | 统计异常标记数 vs 人工确认数，计算准确率 |
| 模型漂移检测 | 每季度 | AI运营 | 对比不同时期的分类分布，检测漂移 |
| 合规符合性 | 每季度 | 合规 | 审核数据处理是否符合隐私保护要求 |
| Skill版本审查 | 每季度 | AI运营 | 确认Prompt/模型版本与最新监管要求匹配 |
