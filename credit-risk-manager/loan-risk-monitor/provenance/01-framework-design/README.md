# 步骤1:框架设计

## 输入
- 8个知识源(监管文件、数据源调研、专家经验、案例萃取)
- 贷后风险监测业务需求

## 过程
- 研究贷后风险监测的标准方法论
- 确定五大监测维度(司法/资产/经营/信用/舆情)
- 设计风险评级矩阵(红色/橙色/黄色/绿色)
- 制定分级处置建议框架
- 确定输出格式为Markdown结构化报告

## 输出
- 监测维度框架文档
- 风险评级矩阵
- 分级处置建议

## 关键决策
- 采用混合型管线,综合运用数据分析(多维度风险扫描)和制度合规(监管要求遵循)
- 确定输出格式为Markdown结构化报告,便于人工阅读和存档
- 设计8条Constraints,含2条合规红线(禁止数据猜测、禁止越权建议)
