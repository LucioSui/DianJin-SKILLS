# 阶段1：框架设计

## 输入
- 企业信贷风险结构化提取业务需求
- GB/T 4754-2017国标行业分类标准文档
- 现有信贷报告风险分析流程文档

## 过程
1. 调研风险点结构化提取方法论
2. 分析企业标签画像体系的完整性与可操作性
3. 设计"信息提取→行业分类→标签填充→风险转化→完整性自查"的核心流程
4. 定义标签体系的五大分类（基础属性/经营特征/风险特征/外部环境/行业分类）
5. 制定风险点结构化转化的六维度框架（风险定义/原文描述/分析方向/数据来源/适用场景/建议关注事项）

## 输出
- 企业标签体系参考 → 已下沉至 `references/tag-system.md`
- 适用场景与建议关注事项规则 → 已下沉至 `references/scenario-rules.md`
- Workflow流程设计 → 已整合至 `SKILL.md`
- 风险点结构化转化规则 → 已整合至 `references/scenario-rules.md`

## 关键决策
- **管线类型**：混合型（数据驱动型为主+制度合规型）
  - 数据驱动特征：风险点结构化提取、标签画像填充、行业分类四级定位、数据可验证规则转化
  - 制度合规特征：GB/T 4754-2017国标行业分类、风险点零遗漏强制要求、合规红线遵循
- **核心原则**：风险点零遗漏、原文原文复制、禁止风险分级、行业分类四级到底
- **标签体系设计**：14类标签分为5大维度，资本属性与组织架构为必填项
