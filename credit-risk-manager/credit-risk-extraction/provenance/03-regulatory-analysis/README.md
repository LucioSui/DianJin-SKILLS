# 阶段3：监管合规分析

## 输入
- 商业银行信贷风险管理监管要求
- 企业信贷授信审批制度
- 风险报告合规性标准

## 过程
1. 分析监管对信贷风险识别与报告的合规要求
2. 提取风险点提取中的关键合规红线（禁止分级、禁止改写、零遗漏）
3. 设计数据溯源机制（每个风险点必须标注数据来源）
4. 制定数据脱敏规则（企业敏感信息处理）
5. 定义免责声明模板，确保输出不构成信贷审批意见

## 输出
- 合规红线清单 → 已整合至 `SKILL.md` 的 Constraints 章节
- 数据脱敏规则 → 已整合至 `references/data-sources-priority.md`
- 免责声明模板 → 已整合至 `assets/risk-extraction-report-template.md`
- 审计日志格式 → 已整合至 `SKILL.md` 的 Audit Trail 章节

## 合规要求要点
1. **风险点零遗漏**：原文有N点风险，输出必须覆盖N点，不得合并或跳过
2. **原文原文复制**：每个风险点的原文描述必须与原文完全一致，严禁缩减、改写或归纳
3. **禁止风险分级**：严禁出现"高风险"、"中风险"、"低风险"等分级表述
4. **数据溯源完整**：每个风险点必须区分已知数据源和可推断数据源
5. **免责声明必需**：输出必须包含"不构成信贷审批意见"、"仅供参考"等声明
