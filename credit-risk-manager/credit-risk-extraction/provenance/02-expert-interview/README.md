# 阶段2：专家访谈与案例萃取

## 输入
- 信贷审批专家经验（风险识别实战案例）
- 常见误判场景汇总
- 行业基准数据（制造业/贸易业/服务业）

## 过程
1. 萃取信贷审批人员在风险点识别中的关键判断逻辑
2. 收集历史案例中的典型风险模式与提取方法
3. 整理常见误判场景及处理规则（如模糊表述替换、风险点拆分）
4. 提炼标签填写策略（何时填"不适用"、何时填"需核实"）
5. 制定风险点结构化转化标准（六维度框架）

## 输出
- 常见误判场景表 → 已整合至 `references/scenario-rules.md`
- 标签填写策略 → 已整合至 `references/tag-system.md`
- 风险点拆分规则 → 已整合至 `references/scenario-rules.md`
- 原文原文复制原则 → 已整合至 `SKILL.md` 的 Constraints 章节

## 专家经验要点
1. **风险点拆分**：若一段风险描述包含多个独立风险（如供应商集中+客户集中），须拆分为多个风险点
2. **标签填写**：原文未提及的标签填"（原文未提及）"，资本属性与组织架构填"（需补充核实）"
3. **禁止改写**：原文描述必须完整复制，任何缩减、归纳、改写均视为错误
4. **模糊词汇替换**：将"相关"、"某些"、"适当"等替换为具体表述或删除
