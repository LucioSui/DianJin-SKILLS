# 置信度评估规则

> data_valid_until: 2027-01-01

## 置信度三因子计算

每个检测点的置信度由以下三因子加权计算：

**置信度 = 证据数量权重(30%) + 证据质量权重(40%) + 多源一致性权重(30%)**

### 证据质量排序

| 质量等级 | 证据类型 | 权重系数 |
|---------|---------|---------|
| 最高 | VLM直接观察原图（如视觉识别装修档次） | 1.0 |
| 高 | LLM基于文本描述分析（如流水结构化数据分析） | 0.8 |
| 中 | 规则匹配（如行业基准对比） | 0.6 |
| 低 | 单一来源信息（无交叉验证） | 0.4 |

### 置信度分级

| 置信度区间 | 分级 | 结论类型 | 输出要求 |
|-----------|------|---------|---------|
| ≥ 0.8 | 高置信度 | 明确结论 | 直接输出"未发现异常"或"存在异常" |
| 0.6-0.8 | 中置信度 | 倾向性结论 | 输出结论 + 标注"建议复核" |
| 0.4-0.6 | 低置信度 | 疑似异常 | 输出"疑似异常" + 详细核实建议 |
| < 0.4 | 极低置信度 | 待核实 | 输出"待核实" + 缺失数据清单 |

## VLM调用策略

1. **默认策略**：图片分析优先用LLM基于描述判断，节省推理成本
2. **触发条件**：当LLM判断置信度 < 70%时，调用VLM分析原图
3. **VLM适用场景**：
   - 装修档次评估
   - 场所规模估算
   - 证件防伪特征识别
   - 设备/存货真实性判断

## 迭代推理机制

每个检测点最多迭代3次：

1. **Think**：规划验证路径，确定需要哪些数据
2. **Check**：执行数据比对，调用VLM/LLM分析
3. **Research**：分析差异，对照误判场景表排除合理例外
4. **循环条件**：置信度 < 0.6 且未达最大迭代次数 → 回到Think
5. **退出条件**：置信度 ≥ 0.6 或达到最大迭代次数3次
