# 阶段3：监管合规分析

## 输入
- 商业银行信贷反欺诈监管要求
- 企业贷款材料真实性核验制度
- 证据链完整性标准

## 过程
1. 分析监管对信贷材料真实性核验的合规要求
2. 提取反欺诈审查中的关键合规红线
3. 设计证据链可追溯机制（每条结论必须标注数据来源文件名及提取字段）
4. 制定数据脱敏规则（个人身份证号、银行账号等敏感信息处理）
5. 定义免责声明模板，确保输出不构成信贷审批意见

## 输出
- 合规红线清单 → 已整合至 `SKILL.md` 的 Constraints 章节
- 数据脱敏规则 → 已整合至 `SKILL.md` 的 Data Sources 章节
- 免责声明模板 → 已整合至 `assets/verification-report-template.md`
- 审计日志格式 → 已整合至 `SKILL.md` 的 Audit Trail 章节

## 合规要求要点
1. **证据链完整**：每条欺诈判断必须对应具体数据来源和比对结果，禁止无依据的主观判断
2. **可解释性要求**：每个检测点须输出"数据来源 → 比对逻辑 → 验证结论"完整推理链
3. **禁止风险分级混淆**：仅描述欺诈事实与异常信号，不输出"高风险/低风险"等主观分级
4. **数据时效性**：数据超过3个月必须在报告开头醒目标注"⚠️ 数据可能已过时"
5. **禁止越权**：不构成任何形式的信贷审批意见、风险定论或决策建议
