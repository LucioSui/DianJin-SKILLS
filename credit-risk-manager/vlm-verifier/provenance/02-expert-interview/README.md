# 阶段2：专家访谈与案例萃取

## 输入
- 信贷审批专家经验（反欺诈实战案例）
- 常见误判场景汇总
- 行业基准数据（餐饮/零售/制造业）

## 过程
1. 萃取信贷审批人员在材料核验中的关键判断逻辑
2. 收集历史案例中的典型欺诈模式与识别方法
3. 整理常见误判场景及处理规则（如装修由房东承担、流水账户非主要收款账户等）
4. 提炼VLM调用策略（何时用LLM描述判断，何时调用VLM分析原图）
5. 制定置信度三因子计算模型（证据数量30% + 证据质量40% + 多源一致性30%）

## 输出
- 常见误判场景表 → 已整合至 `references/cross-validation-matrix.md`
- VLM调用策略 → 已整合至 `references/confidence-rules.md`
- 行业基准参考值 → 已整合至 `references/cross-validation-matrix.md`
- 迭代推理机制（Think→Check→Research循环） → 已整合至 `references/confidence-rules.md`

## 专家经验要点
1. **交叉验证优先**：单一来源信息不得直接得出结论，必须与至少一个独立数据源交叉印证
2. **不确定性标注**：证据不足时标注"待核实"，禁止用模糊证据得出确定性结论
3. **误判排除**：每次检测到异常信号时，必须先对照常见误判场景表排除合理例外情形
4. **成本优化**：图片分析优先用LLM基于描述判断，置信度<70%时再调用VLM分析原图
