# 步骤 1：分析框架设计

## 输入
- 原 SKILL.md（190 行，仅含 name/description/角色定位/核心约束/6步推理/输出模板/质量要求/应用场景/输入参数）
- CLAUDE.md 项目指令
- METHODOLOGY.md 金融级标准

## 过程
1. 读取原 SKILL.md，识别其业务特征：信贷风险逆向思维链生成——给定数据和风险点，构建推理链
2. 判定管线类型：**混合型**（制度合规型 + 数据驱动型）
   - 制度合规型特征：引用监管文件（产业结构调整目录）、产业政策研判、合规判断
   - 数据驱动型特征：财务指标计算、敏感性测算、DSCR 压力测试
3. 判定交互模式：**模式 A（报告生成型）**——输入明确（原始信贷数据+目标风险点），输出为完整 CoT 推理报告，单次调用
4. 编排定位分析：
   - 上游依赖：无强制上游，可独立调用
   - 下游消费：credit-risk-classification（风险分类）、post-loan-management（贷后管理）
5. 提取原 SKILL.md 核心业务规则保护清单（9 项）：
   - ① 无数据时终止输出
   - ② 禁止捏造数据（6 条子规则）
   - ③ 4 场景特定规范（账龄/资本金/短期偿债/关联交易）
   - ④ 6 步推理框架必须覆盖
   - ⑤ 报表口径一致性
   - ⑥ 数据零捏造
   - ⑦ 中间跳板完整
   - ⑧ 正反证据均衡
   - ⑨ 不确定性显式标注
6. 设计 SKILL.md 金融级结构

## 输出
- 管线类型判定报告（混合型）
- 交互模式判定报告（模式 A）
- 编排定位报告
- 核心业务规则保护清单（9 项）
- 详见：[框架设计记录](framework-design.md)（待创建）
