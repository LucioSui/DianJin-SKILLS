# 步骤1: 框架设计

## 输入
- 8个知识源(行业生命周期理论、财务基准数据、欺诈模式案例库、监管政策文件等)
- 行业信贷风险规则生成业务需求

## 过程
- 研究行业信贷风险规则生成的标准方法论
- 确定5大分析维度(行业基本面、产业链风险、信贷风险特征、监管合规要求、资产与抵押物特征)
- 设计规则生成框架(基础核查规则+深度推理规则)
- 制定一票否决条件(I1-I6行业级红线信号)
- 确定输出格式为JSON结构化

## 输出
- 分析维度框架文档
- 规则生成框架设计
- 一票否决条件清单

## 关键决策
- 采用混合型管线,综合运用制度合规(监管要求)和数据分析(行业基准)
- 确定输出格式为JSON结构化,便于下游系统解析
- 设计8条Constraints,包含4条合规红线(禁止收益承诺、禁止数据猜测、数据时效性标注、禁止越权建议)
