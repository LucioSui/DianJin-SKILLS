# 工作流程

```
信息采集 → 持仓诊断 → 缺口测算 → 配置建议 → 报告产出
```

### 第一步：信息采集

通过对话分步收集以下四类信息。每类信息必须确认完毕后再进入下一类。对于用户未提供的可选信息，使用合理默认值并在报告中标注"推断值"。

与用户确认（含默认值）：

| 输入 | 选项 | 默认 |
|------|------|------|
| 客户年龄 | 0-100岁 | 35岁 |
| 家庭结构 | 单身 / 夫妻 / 三口之家 / 多代同堂 | 夫妻 |
| 家庭年收入 | 任意金额 | 30万元 |
| 是否有房贷 | 是 / 否 | 是 |
| 已有保单数量 | 0 / 1-3 / 4+ | 0 |
| 健康状况 | 标准体 / 非标体 / 有既往症 | 标准体 |
| 保费预算 | 占年收入比例 | 10% |
| 核心诉求 | 全面保障 / 重疾优先 / 养老规划 | 全面保障 |

#### 1.1 基础画像（必填）

需收集：年龄、性别、职业（含职业类别 1-6 类）、所在城市（一线/二线/三线）、婚姻状况。

#### 1.2 家庭责任（必填）

需收集：家庭结构（配偶情况、子女数量及年龄、需赡养老人数量及年龄）、子女教育规划（公立/私立/国际）、赡养义务分摊比例。

#### 1.3 财务状况（必填）

需收集：个人/家庭年收入、年支出、负债明细（房贷余额及剩余年限、车贷、其他负债）、流动资产（存款、投资等）、保费预算意愿。

#### 1.4 健康与保单持仓（按需收集）

健康信息：既往病史、体检异常（结节、息肉、指标异常等）、BMI、长期用药情况。
现有保单：逐份记录险种、保额、保障期限、年缴保费、已缴年限、承保公司。

**采集原则：**
- 使用 AskUserQuestion 工具进行结构化采集（如可用）
- 信息不完整时，明确告知用户哪些信息缺失，以及缺失会如何影响分析精度
- 所有推断值必须在最终报告中标注

### 第二步：持仓诊断

对用户现有保单进行系统性诊断。如用户无现有保单，跳过此步骤。

#### 2.1 诊断维度

对每份保单评估以下维度，并给出诊断结论：

**保障充足性**：当前保额 vs 建议保额，缺口比例
**保障重叠**：多份保单保障范围是否重复（如两份百万医疗）
**保障错配**：保障重点是否匹配家庭角色（如经济支柱保额不足、非支柱过度配置）
**性价比**：同等保障条件下保费是否偏高
**保障断层**：保障期限是否存在空窗期

#### 2.2 诊断输出格式

对每个发现，使用以下结构输出：

```
【诊断项】：具体发现
【判断依据】：为什么这是一个问题（引用具体数据）
【建议动作】：保留 / 调整 / 替换 / 补充
```

### 第三步：缺口测算

分四个维度进行缺口测算。每个维度必须展示完整的计算过程。

#### 3.1 寿险缺口（可解释性计算）

**输出格式要求：**

```
┌─ 寿险缺口计算 ─────────────────────────────┐
│                                              │
│  家庭责任总额 =                               │
│    房贷余额           ___万                   │
│  + 其他负债           ___万                   │
│  + 子女教育金         ___万  ← 计算依据：...    │
│  + 父母赡养费         ___万  ← 计算依据：...    │
│  + 家庭生活费(N年)    ___万  ← 年支出×N年      │
│  ─────────────────────────                   │
│  = 总需求             ___万                   │
│                                              │
│  可抵扣资产 =                                 │
│    流动资产           ___万                   │
│  + 已有寿险保额       ___万                   │
│  ─────────────────────────                   │
│  = 总可抵扣           ___万                   │
│                                              │
│  ★ 寿险缺口 = ___万 - ___万 = ___万           │
│  ★ 建议保额：___万（缺口上浮10%安全边际）       │
│                                              │
└──────────────────────────────────────────────┘
```

每一个数字都必须附带来源说明（用户提供值 / 参考标准 / 推断值）。

#### 3.2 重疾缺口（可解释性计算）

```
重疾缺口 = 治疗费用 + 康复费用 + 收入损失(3-5年) - 已有重疾保额

每项计算依据：
- 治疗费用：参考城市级别标准（详见 reference.md）
- 康复费用：治疗费用的 30%-50%
- 收入损失：年收入 × 3年（保守）或 5年（充分）
```

说明选用的参数值及其理由。

#### 3.3 医疗缺口

评估现有医疗保障覆盖范围，判断是否需要补充：
- 社保覆盖上限与实际医疗费用的差距
- 免赔额承受能力
- 就医需求层次（普通/特需/国际）

#### 3.4 意外缺口

根据职业类别和生活方式评估：
- 职业风险等级 → 对应保额建议
- 通勤/出差频率 → 是否需要额外交通意外保障
- 是否需要含猝死保障

**每个维度的输出都必须包含：**
1. 计算过程（公式 + 代入数值）
2. 参数来源（用户提供 / 行业标准 / 推断）
3. 最终结论（缺口金额 + 建议保额）

详细的计算参数和参考数据见 [reference.md](reference.md)。

### 第四步：配置建议

#### 4.1 建议原则

按以下优先级排序并说明理由：
1. **先保障后理财** — 优先配置保障型产品
2. **先支柱后家属** — 优先保障家庭经济支柱
3. **先高杠杆后低杠杆** — 有限预算优先配置杠杆率高的险种
4. **保额优先于保障期限** — 预算不足时缩短期限保住保额

#### 4.2 缺口配置决策映射表

| 缺口维度 | 是否存在缺口 | 缺口等级 | 建议动作 | 优先级 |
|---------|------------|---------|---------|--------|
| 寿险缺口 | 是 / 否 | 高 / 中 / 低 | 立即补充定期寿险 / 逐步完善 / 维持现有 | 高 / 中 / 低 |
| 重疾缺口 | 是 / 否 | 高 / 中 / 低 | 优先配置重疾险 / 预算内补充 / 维持现有 | 高 / 中 / 低 |
| 医疗缺口 | 是 / 否 | 高 / 中 / 低 | 立即配置百万医疗 / 升级产品 / 维持现有 | 高 / 中 / 低 |
| 意外缺口 | 是 / 否 | 高 / 中 / 低 | 补充综合意外 / 增加交通意外 / 维持现有 | 高 / 中 / 低 |
| 养老缺口 | 是 / 否 | 高 / 中 / 低 | 优先配置养老年金 / 增额终身寿 / 维持现有 | 中 / 低 / 低 |

#### 4.3 可解释性建议输出

对每条建议使用以下推理链结构：

```
【建议】：配置定期寿险，保额300万，保至60岁

【推理链】：
  ① 事实：用户为单收入家庭经济支柱，房贷余额100万，家庭年支出20万
  ② 计算：寿险缺口 = 100 + 200(10年生活费) + 40(赡养) - 50(资产) = 290万
  ③ 判断：缺口290万，上浮10%安全边际 → 建议保额300万
  ④ 选型：定期寿险性价比最优，保至60岁覆盖主要责任期
  ⑤ 预算：预估年缴约4,500元，占年收入1.1%，在合理范围内

【替代方案】：若预算紧张，可选保至55岁，保费降低约30%
```

#### 4.4 预算分配

给出整体预算分配方案，展示各险种占比及理由：

```
推荐年度保费预算：___元（占年收入___%）

预算分配：
  重疾险：___元（占比___%）← 保费占比最大，需优先保障
  寿险  ：___元（占比___%）← 定期寿险杠杆率高
  医疗险：___元（占比___%）← 费用较低但必须配置
  意外险：___元（占比___%）← 费用最低，保障杠杆最高
```

### 第五步：报告产出

#### 5.1 输出格式

使用 docx skill 或 pdf skill 生成专业报告文档。报告结构参考 [references/output-template.md](references/output-template.md)。

#### 5.2 报告核心章节

1. **客户画像摘要** — 基础信息一览表
2. **现有保障诊断** — 持仓分析结论（如有保单）
3. **保障缺口分析** — 四维缺口测算（含完整计算过程）
4. **配置建议方案** — 带推理链的逐条建议
5. **预算分析** — 保费预算与收入占比
6. **实施路径** — 按优先级排列的投保步骤
7. **附录：计算说明** — 所有公式和参考数据来源

#### 5.3 报告要求

- 所有数字必须有来源标注
- 推断值使用"*"标记并在附录说明
- 计算过程使用表格或公式块展示，确保可读性
- 报告末尾附免责声明：本报告为保障分析参考，不构成具体产品推荐，投保前请咨询专业保险顾问
