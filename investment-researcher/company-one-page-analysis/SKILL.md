# 上市公司一页纸分析技能

## 技能定位

本技能面向**投资经理、研究员、基金经理**，提供上市公司快速分析工具。在一页纸篇幅内，聚焦公司最核心的投资信息，帮助快速判断公司投资价值和风险，适合晨会汇报、快速决策、标的初筛等场景。

**核心优势**：
- ✅ **gildata-aidata 专业数据源**：所有财务数据、估值数据优先通过 gildata-aidata 服务获取
- ✅ **精炼高效**：五大核心维度，一页纸篇幅，直击投资要点
- ✅ **快速决策**：适合晨会汇报、标的跟踪、快速初筛

与 `company-deep-analysis` 的区别：
- **深度分析**：七大维度，全面深入，适合深度研究和首次覆盖
- **一页纸分析**：五大核心维度，精炼简洁，适合快速决策和标的跟踪

## 数据获取策略

### 核心数据源

**核心原则：所有财务数据、估值数据优先通过 gildata-aidata 服务获取。**

| 数据源 | 用途 | 优先级 |
|--------|------|--------|
| **gildata-aidata** | 财务报表、财务分析指标、估值数据、一致预期 | **第一优先级** |
| `web_search` | 公司基本信息、最新新闻、行业政策、风险信息 | 补充数据源 |
| `asset-service` | 企业工商基本信息、司法风险（如需） | 辅助数据源 |

### 数据获取优先级

1. **财务数据**：优先 gildata-aidata `FinancialStatement` / `FinancialAnalysis`
2. **估值数据**：优先 gildata-aidata `StockValueAnalysis`
3. **基础信息**：web_search 或 asset-service
4. **新闻资讯**：web_search 补充

## 分析框架（一页纸五大维度）

### 一、公司画像（基本信息）

**数据获取**：
```
使用 web_search 查询：
- "[公司名称] 公司简介 股票代码 所属行业"

或使用 asset-service query_industry_epinfo 查询：
- "[公司名称]" 获取企业基本信息
```

**核心信息**：
- 公司全称、股票代码
- 所属行业/板块
- 市值、股价
- 实控人/大股东

### 二、核心财务（关键指标）

**数据获取**：
```
使用 gildata-aidata FinancialStatement 查询：
- "{公司名} 2022 2023 2024 年报 利润表"

使用 gildata-aidata FinancialAnalysis 查询：
- "{公司名} 2022 2023 2024 毛利率 净利率 ROE ROIC"
```

**核心指标**（表格呈现）：
| 指标 | 2022 | 2023 | 2024 | 趋势 |
|------|------|------|------|------|
| 营收（亿元） | | | | |
| 净利润（亿元） | | | | |
| 毛利率（%） | | | | |
| ROE（%） | | | | |
| 经营现金流（亿元） | | | | |

*数据来源：gildata-aidata FinancialStatement/FinancialAnalysis*

### 三、业务亮点（投资逻辑）

**数据获取**：
```
使用 web_search 查询：
- "[公司名称] 核心业务 竞争优势 投资亮点 2025 2026"
```

**分析要点**：
- 核心业务/主营产品
- 核心竞争优势（1-3 点）
- 近期催化剂/增长点

### 四、估值水平（横向对比）

**数据获取**：
```
使用 gildata-aidata StockValueAnalysis 查询：
- "{公司名} PE PB PS EV 股息率"

使用 web_search 查询：
- "[行业名称] 估值 可比公司 PE"
```

**核心信息**：
- 当前 PE、PB
- 历史估值分位（如可得）
- 与行业/可比公司对比
- 估值判断（低估/合理/高估）

*数据来源：gildata-aidata StockValueAnalysis*

### 五、主要风险（风险提示）

**数据获取**：
```
使用 web_search 查询：
- "[公司名称] 风险 负面 监管"

使用 asset-service 查询（如需）：
- 行政处罚、司法风险等
```

**风险要点**：
- 列出 2-4 个核心风险点
- 按重要性排序

## 执行流程

### 步骤 1：确认分析对象
- 从用户输入中提取公司名称
- 使用 `web_search` 搜索确认公司全称和股票代码
- 获取所属行业、市值等基础信息

### 步骤 2：获取核心财务数据
```
调用 gildata-aidata FinancialStatement：
query: "{公司名} 2022 2023 2024 年报 利润表 资产负债表 现金流量表"

调用 gildata-aidata FinancialAnalysis：
query: "{公司名} 2022 2023 2024 毛利率 净利率 ROE ROIC 资产负债率"
```

### 步骤 3：获取估值数据
```
调用 gildata-aidata StockValueAnalysis：
query: "{公司名} PE PB PS EV 股息率"
```

### 步骤 4：获取业务信息
```
调用 web_search：
query: "[公司全称] 核心业务 竞争优势 投资亮点 2025 2026"
```

### 步骤 5：获取风险信息
```
调用 web_search：
query: "[公司全称] 风险 负面 监管 诉讼"

调用 asset-service（如需）：
key_word: "[公司全称]"
```

### 步骤 6：撰写一页纸报告
按照上述"报告输出格式"撰写报告，确保：
- 严格控制篇幅在一页纸内（约 800-1200 字）
- 关键数据使用表格呈现
- 语言精炼，直击核心
- 结论明确

### 步骤 7：报告交付

**重要**：报告生成后直接在对话中输出完整的 Markdown 报告内容。

**执行方式**：
```
直接将完整的一页纸 Markdown 报告内容在对话中输出
```

**注意**：
- **不需要发送文件**，只需在对话中直接显示
- 报告应简洁明了，适合快速阅读
- 核心数据突出显示

## 注意事项

1. **篇幅控制**
   - 严格控制在"一页纸"篇幅内
   - 避免冗长描述，每点 1-2 句话
   - 优先使用表格呈现数据

2. **数据时效性**
   - 使用最新可得的财务数据
   - 标注数据截止日期
   - 估值数据使用最新行情

3. **聚焦核心**
   - 只呈现最关键的信息
   - 避免细节堆砌
   - 突出投资逻辑主线

4. **客观中立**
   - 保持客观，避免过度推荐
   - 风险提示必须充分
   - 明确说明"不构成投资建议"

5. **报告交付要求**
   - **直接在对话中显示完整报告内容**
   - 不需要发送文件
   - 使用 Markdown 格式，确保可读性

6. **数据准确性**
   - 财务数据需标注来源和报告期，优先使用 gildata-aidata 获取
   - 估值数据为最新值
   - 不编造数据，数据暂缺时明确标注

## 特殊情况处理

### 情况 1：数据获取失败
- 尝试使用替代数据源
- 如仍无法获取，标注"数据暂缺"
- 不编造数据

### 情况 2：非上市公司
- 说明数据局限性
- 使用 asset-service 获取基础工商信息
- 调整报告格式，弱化财务和估值部分

### 情况 3：境外公司
- gildata-aidata 港股/美股数据可能有限
- 结合 web_search 获取信息
- 使用对应的港股/美股 gildata-aidata 工具
- 在报告中说明数据覆盖范围

### 情况 4：用户要求更详细
- 如用户表示"太简单了，要更详细"
- 建议调用 `company-deep-analysis` 技能进行深度分析
- 或在一页纸框架内适当扩展各部分内容
