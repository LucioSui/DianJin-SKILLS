# 行业深度分析技能

## 技能定位

本技能面向**专业研究员、投资机构、产业分析师**，提供全方位行业深度分析。通过调用多个专业金融数据工具和行业数据库，从八大维度系统性地分析行业投资价值、竞争格局和发展趋势，生成机构级行业深度研究报告。

## 触发场景

当用户出现以下任一表述时触发本技能：
- "对 XX 行业做深度分析"
- "分析 XX 行业"
- "XX 行业深度研究"
- "研究 XX 行业的投资价值"
- "XX 行业全面分析"
- "帮我深度研究一下 XX 行业"
- "XX 行业投资机会分析"
- "XX 行业发展趋势分析"
- 其他要求对行业进行多维度深度分析的请求

**适用行业类型**：
- A 股/港股/美股相关行业（如白酒行业、新能源汽车行业、半导体行业等）
- 新兴产业（如人工智能、云计算、生物医药等）
- 传统行业（如房地产、银行、保险、煤炭等）
- 细分赛道（如光伏逆变器、锂电隔膜、CXO 等）

## 数据获取策略

本技能采用多工具协同的数据获取策略，确保数据权威性和全面性：

### 核心数据源

**核心原则：所有行业数据、市场规模、竞争格局、财务估值数据优先通过 gildata-aidata 服务获取。**

| 数据源 | 用途 | 优先级 |
|--------|------|--------|
| **gildata-aidata** | 行业概览、市场规模、竞争格局、行业趋势、成分股筛选、财务估值 | **第一优先级** |
| `web_search` | 行业政策信息、最新研究报告、国际对比数据、补充信息 | 补充数据源 |
| `asset-service` | 行业内主要企业工商信息、股权穿透、基础信息 | 辅助数据源 |

### 数据获取优先级

1. **行业数据与规模**：优先 gildata-aidata `IndustryOverview` / `IndustryMarketScale`
2. **竞争格局与成分股**：优先 gildata-aidata `IndustryCompetition` / `StockPool`
3. **行业趋势与研报**：优先 gildata-aidata `IndustryTrend` / `ResearchReport`
4. **企业信息**：gildata-aidata `CompanyProfile` 或 asset-service
5. **政策与宏观**：web_search 补充

## 分析框架（八大维度）

### 一、行业概况

**数据获取**：
```
使用 gildata-aidata IndustryOverview 查询：
- "[行业名称] 行业定义 分类 发展历程 生命周期"

使用 web_search 查询：
- "[行业名称] 行业介绍 基本概念 产业链 宏观环境"
```

**分析要点**：
1. **行业定义与分类**
   - 行业的明确定义
   - 行业细分领域/子行业
   - 行业在国民经济中的分类（如申万行业分类）

2. **行业发展历程**
   - 行业起源与发展阶段
   - 重要历史节点和里程碑
   - 当前所处生命周期阶段（导入期/成长期/成熟期/衰退期）

3. **行业特征**
   - 行业商业模式
   - 行业盈利模式
   - 行业周期性特征

### 二、市场规模

**数据获取**：
```
使用 gildata-aidata IndustryMarketScale 查询：
- "[行业名称] 市场规模 市场空间 增速 历史数据 预测"

使用 web_search 查询：
- "[行业名称] 市场规模 2024 2025 2026 预测 增长率 行业报告"
```

**分析要点**：
1. **当前市场规模**
   - 市场容量（金额/数量）
   - 近 3-5 年市场规模变化
   - 市场规模增长率

2. **市场驱动因素**
   - 需求驱动因素
   - 供给驱动因素
   - 政策驱动因素

3. **未来市场空间**
   - 未来 3-5 年市场规模预测
   - 市场增长率预测
   - 市场天花板判断

4. **渗透率分析**（如适用）
   - 当前渗透率水平
   - 渗透率提升空间
   - 渗透率提升速度

### 三、产业链分析

**数据获取**：
```
使用 gildata-aidata IndustryChain 查询：
- "[行业名称] 产业链 上游 下游 价值链"

使用 web_search 查询：
- "[行业名称] 产业链图谱 价值链分布 关键节点"
```

**分析要点**：
1. **产业链结构**
   - 上游：原材料、设备、技术等
   - 中游：制造、加工、组装等
   - 下游：应用、渠道、终端客户等

2. **价值链分布**
   - 各环节附加值分布
   - 利润分布情况
   - 议价能力对比

3. **产业链关键环节**
   - 卡脖子环节
   - 高壁垒环节
   - 高成长环节

4. **产业链趋势**
   - 垂直整合趋势
   - 产业链转移趋势
   - 供应链安全考量

### 四、竞争格局

**数据获取**：
```
使用 gildata-aidata IndustryCompetition 查询：
- "[行业名称] 竞争格局 市场集中度 CR3 CR5 主要企业"

使用 gildata-aidata StockPool 查询：
- 筛选行业内上市公司及财务数据

使用 web_search 查询：
- "[行业名称] 市场份额 竞争态势 龙头企业"
```

**分析要点**：
1. **市场集中度**
   - CR3/CR5/CR10（前 3/5/10 企业集中度）
   - 行业集中度变化趋势
   - 与发达国家对比

2. **主要参与者**
   - 行业龙头及市场份额
   - 主要上市公司列表
   - 非上市主要企业

3. **竞争态势**
   - 竞争格局类型（完全竞争/垄断竞争/寡头垄断/完全垄断）
   - 竞争焦点（价格/技术/品牌/渠道）
   - 竞争激烈程度

4. **企业对比分析**
   - 主要企业财务指标对比
   - 主要企业竞争优势对比
   - 主要企业战略定位对比

### 五、行业趋势

**数据获取**：
```
使用 gildata-aidata IndustryTrend 查询：
- "[行业名称] 发展趋势 技术趋势 未来方向 创新"

使用 web_search 查询：
- "[行业名称] 发展趋势 2025 2026 技术创新 需求变化 商业模式"
```

**分析要点**：
1. **技术发展趋势**
   - 核心技术演进方向
   - 新技术/新工艺应用
   - 技术迭代速度

2. **需求变化趋势**
   - 消费需求变化
   - 客户偏好变化
   - 新兴需求涌现

3. **产品/服务趋势**
   - 产品升级方向
   - 服务创新方向
   - 产品差异化趋势

4. **商业模式趋势**
   - 商业模式创新
   - 数字化转型
   - 平台化/生态化趋势

### 六、进入壁垒

**数据获取**：
```
使用 web_search 查询：
- "[行业名称] 进入壁垒 行业门槛 竞争壁垒"

使用 gildata-aidata IndustryBarrier 查询（如可用）：
- "[行业名称] 技术壁垒 资质壁垒 资金壁垒"
```

**分析要点**：
1. **技术壁垒**
   - 核心技术门槛
   - 专利保护情况
   - 技术积累要求

2. **资金壁垒**
   - 初始投资规模
   - 运营资金需求
   - 融资难度

3. **资质/牌照壁垒**
   - 行业准入要求
   - 资质认证要求
   - 牌照获取难度

4. **品牌/渠道壁垒**
   - 品牌建设周期
   - 渠道建设难度
   - 客户转换成本

5. **规模/人才壁垒**
   - 规模经济效应
   - 核心人才获取难度
   - 管理经验要求

### 七、风险因素

**数据获取**：
```
使用 web_search 查询：
- "[行业名称] 风险因素 行业风险 投资风险"

使用 gildata-aidata IndustryRisk 查询（如可用）：
- "[行业名称] 政策风险 市场风险 技术风险"
```

**分析要点**：
1. **政策风险**
   - 产业政策变化
   - 监管政策趋严
   - 贸易政策影响

2. **市场风险**
   - 需求波动风险
   - 价格波动风险
   - 竞争加剧风险

3. **技术风险**
   - 技术迭代风险
   - 技术路线不确定性
   - 技术被替代风险

4. **经营风险**
   - 原材料价格波动
   - 人力成本上升
   - 环保合规成本

5. **财务风险**
   - 行业整体负债水平
   - 现金流状况
   - 融资环境变化

### 八、投资机会

**数据获取**：
```
使用 gildata-aidata StockPool 查询：
- 筛选行业内优质上市公司及估值数据

使用 gildata-aidata ResearchReport 查询：
- "[行业名称] 投资建议 标的推荐 研报观点"

使用 web_search 查询：
- "[行业名称] 投资机会 细分赛道 估值分析"
```

**分析要点**：
1. **细分领域机会**
   - 高成长细分赛道
   - 被低估的细分领域
   - 新兴机会点

2. **投资标的筛选**
   - 行业龙头
   - 高成长标的
   - 估值合理标的

3. **投资时点判断**
   - 行业周期位置
   - 估值水平评估
   - 催化剂因素

4. **投资策略建议**
   - 长期配置价值
   - 阶段性机会
   - 风险提示

### 步骤 7：报告撰写
按照上述"报告输出格式"撰写完整报告，确保：
- 数据标注来源
- 关键数据使用表格呈现
- 分析逻辑清晰
- 结论明确
- 风险提示充分

### 步骤 8：报告交付

**重要**：报告生成后直接在对话中输出完整的 Markdown 报告内容，让用户可以立即阅读。

**执行方式**：
```
直接将完整的 Markdown 报告内容在对话中输出（使用 Markdown 格式渲染）
```

**注意**：
- **不需要发送文件**，只需在对话中直接显示完整报告内容
- 报告生成后，可以在对话中提供核心结论摘要，帮助用户快速把握要点
- 如报告过长（超过 15000 字），可在对话中先输出执行摘要和核心章节，再输出完整报告

### 步骤 9：质量检查
- 核对关键数据准确性
- 检查逻辑一致性
- 确保八大维度全覆盖
- 验证风险提示充分性

## 数据来源优先级

| 数据类型 | 第一优先级 | 第二优先级 | 第三优先级 |
|---------|-----------|-----------|-----------|

## 注意事项

1. **数据时效性**
   - 优先使用最新数据（2024 年、2025 年、2026 年）
   - 标注数据截止日期
   - 对于预测数据明确说明是预测值

2. **数据交叉验证**
   - 关键行业数据从多个来源验证
   - 发现数据冲突时，以权威机构数据为准
   - 在报告中标注数据来源

3. **专业性与客观性**
   - 使用专业术语，但避免过度晦涩
   - 保持客观中立，避免过度乐观/悲观
   - 明确区分事实和观点

4. **风险提示**
   - 风险揭示必须充分
   - 不提供确定性收益承诺
   - 明确说明"不构成投资建议"

5. **工具使用优先级**
   - 行业数据：优先 **gildata-aidata** 相关工具
   - 企业信息：优先 **gildata-aidata** StockPool 或 asset-service
   - 行业趋势/政策：可结合 web_search
   - 避免过度依赖单一数据源

6. **报告交付要求**
   - **直接在对话中显示完整报告内容**
   - 不需要发送文件
   - 对话显示使用标准 Markdown 格式

7. **行业边界界定**
   - 对于宽泛行业（如"科技行业"），需要进一步细化
   - 对于细分行业，需要说明与所属行业的关系
   - 明确行业统计口径

## 特殊情况处理

### 情况 1：数据获取失败
如某个工具调用失败：
- 尝试使用替代工具
- 如仍无法获取，在报告中说明"数据暂缺"
- 不编造数据

### 情况 2：行业定义模糊
如用户输入的行业名称过于宽泛（如"科技行业"）：
- 主动与用户确认具体细分领域
- 或自行界定分析范围并在报告中说明
- 建议聚焦到具体细分赛道

### 情况 3：新兴行业数据不足
如分析新兴行业（如 AI 大模型、低空经济等）：
- 说明数据局限性
- 更多依赖网络搜索获取信息
- 在报告中说明数据覆盖范围

### 情况 4：用户要求简化
如用户表示"不用那么复杂"：
- 保留核心分析框架
- 精简每个维度的分析深度
- 优先保留市场规模、竞争格局、趋势、机会四个核心维度
