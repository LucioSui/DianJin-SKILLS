# 全球财经要闻简报技能

## 技能定位

本技能用于**自动抓取全球最新财经要闻**，通过 `gildata-aidata` 金融数据服务获取结构化数据与舆情，智能筛选高影响力事件，生成**结构化简要总结报告**。适用于用户快速获取每日财经热点、市场动态和投资机会参考。

**核心特点**：
- 自动捕获最新财经资讯（24-72 小时内）
- 智能筛选高影响力事件
- 结构化分类汇总
- 简洁易懂的简报格式

## 执行流程

### Step 1：确定搜索范围和时间

**1.1 获取当前日期**

调用 `finx time` 获取当前日期，用于确定新闻时间范围。

```bash
finx time
```

**1.2 确定搜索关键词**

根据用户输入提取核心关键词：
- 用户指定行业/主题 → 使用指定关键词
- 用户未指定 → 使用"今日财经热点 宏观 市场"等通用关键词

**1.3 确定时间范围**

- 默认搜索最近 24-72 小时内的新闻
- 用户指定时间 → 按用户要求调整

---

### Step 2：使用 gildata-aidata 服务查询数据

**核心原则**：优先使用 `gildata-aidata` 服务中的专业金融数据工具，确保数据的权威性、结构化和时效性。

**2.1 舆情与新闻检索**
使用 `gildata-aidata` 的舆情工具获取多维度财经资讯：

| 数据维度 | 推荐工具 | 查询示例 (query 参数) |
|----------|----------|----------------------|
| 全网热点 | `NewsInfoList` | "今日财经热点 重大事件" |
| 宏观政策 | `MacroNewslist` | "今日宏观政策 央行 财政部 经济数据" |
| 政策会议 | `PolicyMeetingsList` | "最新政策会议 经济金融" |
| 官员讲话 | `OfficialSpeechEventList` | "央行行长 财政部官员 重要讲话" |
| 市场动态 | `MarketNewslist` | "今日股市 债市 汇市 大宗商品动态" |
| 行业新闻 | `IndustryNewslist` | "半导体 新能源 AI 医药 行业最新进展" |
| 产业快讯 | `IndustryNewsFlash` | "今日产业快讯 行业要闻" |
| 股票舆情 | `StockNewslist` | "科技股 新能源龙头 财报 业绩" |
| 债券舆情 | `SecurityNewslist` | "信用债 违约 评级调整" |
| 基金舆情 | `FundNewsList` | "公募基金 基金经理 调仓" |

**2.2 行情与市场数据检索**
获取主要市场指数行情、资金流向与宏观数据：

| 数据维度 | 推荐工具 | 查询示例 (query 参数) |
|----------|----------|----------------------|
| A股指数 | `IndexDailyQuote` | "上证指数 深证成指 创业板指 今日行情" |
| 港股指数 | `HKDailyIndexQuote` | "恒生指数 恒生科技 今日行情" |
| 美股指数 | `USStockDailyQuotes` | "道琼斯 标普500 纳斯达克 今日行情" |
| 市场统计 | `StockMarketTradeStats` | "今日A股成交量 成交额 涨跌家数" |
| 资金流向 | `StockMarketCapitalFlow` | "今日北向资金 南向资金流向" |
| 宏观数据 | `MacroIndustryEDB` | "最新GDP CPI PMI 社融 工业企业利润" |
| 研报观点 | `MacroeconomicAnalysisViewpoints` | "宏观研报观点 经济展望" |

**2.3 查询策略**
- **并行查询**：对多个维度的工具发起并行请求，提高效率。
- **query 构造**：使用简洁明确的关键词组合，如 `"今日 宏观 政策"` 或 `"2026-04-28 上证指数 行情"`。可加入日期提高准确性。
- **数据交叉验证**：重要数据（如指数涨跌、宏观数据）可通过多个工具或 `web_search` 进行验证。
- **Fallback 机制**：若 `gildata-aidata` 返回数据不足或无法覆盖特定海外细节，可调用 `web_search` 补充。

### Step 3：新闻筛选与分类

**3.1 筛选标准**

| 维度 | 标准 |
|------|------|
| **时效性** | 优先 24 小时内，最长不超过 72 小时 |
| **影响力** | 政策类 > 宏观数据 > 行业事件 > 公司动态 |
| **热度** | 多家媒体报导 > 单一来源 |
| **相关性** | 与用户指定主题匹配度高 |

**3.2 分类体系**

将筛选后的新闻按以下类别分类：

| 类别 | 包含内容 |
|------|----------|
| **宏观事件** | 国家政策、经济数据、央行决策、地缘政治 |
| **市场表现** | 股市、债市、汇市、大宗商品行情 |
| **行业动态** | 行业政策、供需变化、价格波动、技术突破 |
| **公司新闻** | 财报发布、并购重组、高管变动、重大合同 |
| **投资机会** | 分析师观点、机构研报、热点主题 |

**3.3 重要性评级**

对每条新闻进行重要性评级：

| 评级 | 标准 | 标识 |
|------|------|------|
| **S 级** | 影响全球/全国市场的重大事件 | 🔴 |
| **A 级** | 影响行业/板块的重要事件 | 🟠 |
| **B 级** | 影响部分公司的常规事件 | 🟡 |
| **C 级** | 一般性资讯 | 🟢 |

### Step 5：特殊场景处理

**场景一：用户指定行业/主题**

当用户指定特定行业或主题时（如"半导体行业新闻"、"新能源汽车最新动态"）：

1. 提取用户指定的关键词
2. 针对性搜索该领域新闻（如 `IndustryNewslist`）
3. 报告结构保持不变，但内容聚焦指定领域
4. 增加该领域的深度分析

**场景二：用户要求定时推送**

当用户要求每日/每周定时推送财经简报时：

1. 使用 `cron` 工具创建定时任务
2. 任务 payload 中包含调用本技能的指令
3. 设置合适的推送时间（如交易日早 8 点）

示例：
```bash
cron --action add --taskName "每日财经简报" --description "每个交易日早晨生成全球财经要闻简报" --schedule '{"triggerType": "CRON", "cron": "0 0 8 * * MON-FRI"}' --payload '{"userInput": "生成一份最近 24 小时的全球财经要闻简报，包含核心头条、市场表现、宏观政策、行业动态、投资机会和风险提示。"}'
```

**场景三：重大突发事件**

当检测到重大突发事件（如战争、金融危机、政策突变）时：

1. 在报告开头增加"⚠️ 突发重大事件"板块
2. 详细说明事件背景、进展和潜在影响
3. 增加风险提示的权重
4. 建议用户密切关注后续发展

**场景四：市场休市/节假日**

当遇市场休市或节假日时：

1. 在报告中说明休市情况
2. 汇总休市期间的重要新闻
3. 提示下周开市后的关注要点

## 注意事项

1. **数据时效性**：行情数据需标注具体时间，避免误导用户
2. **客观中立**：分析应基于事实，避免主观臆断和情绪化表达
3. **风险充分**：必须提示主要风险点，尤其是重大不确定性事件
4. **免责声明**：每份报告必须包含免责声明
5. **简洁优先**：简报控制在 1500 字以内为佳，重点突出
6. **来源权威**：优先使用 `gildata-aidata` 金融数据服务
7. **多源验证**：重要新闻建议多数据源交叉验证
8. **敏感信息**：涉及地缘政治等敏感话题，保持客观陈述
9. **用户定制**：根据用户历史偏好调整报告详细程度和关注领域
10. **Fallback 机制**：当 `gildata-aidata` 无法获取某些数据时，自动降级使用 `web_search`
