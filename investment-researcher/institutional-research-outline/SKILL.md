# 机构调研大纲生成技能

## 技能定位

本技能用于**生成上市公司机构调研前的结构化调研大纲**，帮助研究员系统性地准备调研问题，覆盖财务、业务、战略、风险等关键维度。**以 gildata-aidata 金融数据服务为核心数据源**，确保财务数据、估值数据、同业对比的专业性和准确性。

**核心优势**：
- ✅ **gildata-aidata 专业数据源**：所有财务数据、估值数据、一致预期优先通过 gildata-aidata 服务获取
- ✅ **分析框架系统**：覆盖基本面、同业对比、历史问题、核心问题四大模块
- ✅ **问题设计专业**：10-12 个针对性调研问题，避免公开信息重复
- ✅ **输出结构化**：标准化 Markdown 格式报告，直接在对话中展示完整内容

**输出格式**：
- 默认生成结构化 Markdown 格式报告
- **报告内容直接在对话中展示，无需发送文件**
- 如用户明确要求 Word 文档，可另行处理

## 执行流程

### Step 1：获取公司基本信息

**1.1 提取公司名称/股票代码**

从用户输入中识别目标上市公司：
- 公司全称/简称（如"五粮液"、"茅台"）
- 股票代码（如"000858"、"600519"）

**1.2 通过 gildata-aidata 获取财务与估值数据**

**核心原则：所有财务数据、估值数据、一致预期优先通过 gildata-aidata 服务获取。**

| 数据类别 | gildata-aidata 工具 | query 参数示例 |
|---------|-------------------|---------------|
| **财务报表** | `FinancialStatement` | "五粮液 2024 年年报 利润表 资产负债表 现金流量表" |
| **财务分析指标** | `FinancialAnalysis` | "五粮液 2024 年 毛利率 净利率 ROE ROIC 资产负债率 应收账款周转率 存货周转率" |
| **主营构成** | `MainOperIncData` | "五粮液 2024 年 分产品 分行业 分地区 收入 成本 毛利率" |
| **一致预期** | `ConsensusExpectation` | "五粮液 2024 2025 2026 营收 净利润 EPS 一致预期" |
| **估值数据** | `StockValueAnalysis` | "五粮液 PE PB PS EV 股息率" |
| **机构评级** | `InstitutionalRating` | "五粮液 机构评级 评级分数" |
| **财报点评参考** | `AShareFinancialReportReview` | "五粮液 2024 年年报 业绩点评" |

**港股公司**：使用 `HKFinancialStatement`、`HKStockValueAnalysis`、`HKMainOperIncData`

**美股公司**：使用 `USFinancialStatement`、`USValuationMetrics`、`USFinancialAnalysis`

**1.3 通过 web_search 获取补充信息**

用于 gildata-aidata 无法覆盖的信息：
- 公司基本信息（成立时间、员工规模、总部地点等）
- 历史调研记录/投资者关系活动记录
- 最新新闻舆情、行业动态

```bash
web_search --query "{公司名称} 公司简介 主营业务 行业地位" --numResults 10
web_search --query "{公司名称} 投资者关系活动记录 机构调研 问答 2025 2026" --numResults 10
```

### Step 2：基本面分析

**2.1 公司概况**

整理以下信息：
- 公司全称、股票代码、上市交易所
- 成立时间、上市时间
- 主营业务及产品
- 行业分类及细分领域
- 市值、员工规模

**2.2 财务分析（通过 gildata-aidata 获取）**

分析以下核心指标（最近 3 年 + 最新一期）：

| 指标类别 | 具体指标 | 数据来源 |
|----------|----------|----------|
| **成长性** | 营收增速、净利润增速、扣非净利润增速 | `FinancialStatement` |
| **盈利能力** | 毛利率、净利率、ROE、ROIC | `FinancialAnalysis` |
| **营运能力** | 应收账款周转率、存货周转率、总资产周转率 | `FinancialAnalysis` |
| **偿债能力** | 资产负债率、流动比率、速动比率、利息保障倍数 | `FinancialAnalysis` |
| **现金流** | 经营性现金流净额、自由现金流、现金流/净利润比值 | `FinancialStatement` |

**2.3 业务分析（通过 gildata-aidata 获取）**

- 主营业务收入构成（按产品/地区/渠道）→ `MainOperIncData`
- 核心产品/服务及竞争优势
- 主要客户及供应商集中度
- 产能及利用率情况
- 研发投入及核心技术

**2.4 行业地位**

- 市场份额及排名
- 主要竞争对手
- 行业集中度（CR3/CR5）
- 公司在产业链中的位置（上游/中游/下游）

---

### Step 3：同业对比分析

**3.1 确定可比公司**

选择 3-5 家同行业上市公司作为对比对象，选择标准：
- 同一细分行业
- 业务模式相似
- 市值规模相当（或行业龙头）
- A 股/港股/美股优先

**3.2 通过 gildata-aidata 获取对比数据**

对目标公司和可比公司分别调用以下工具：

| 对比维度 | 具体指标 | gildata-aidata 工具 |
|----------|----------|-------------------|
| **估值水平** | PE(TTM)、PB、PS、PEG、EV/EBITDA | `StockValueAnalysis` |
| **成长性** | 营收 3 年 CAGR、净利润 3 年 CAGR | `FinancialStatement` |
| **盈利能力** | 毛利率、净利率、ROE | `FinancialAnalysis` |
| **运营效率** | 存货周转天数、应收账款周转天数 | `FinancialAnalysis` |
| **财务健康** | 资产负债率、经营性现金流/营收 | `FinancialAnalysis` |

**3.3 对比结论**

- 公司在同业中的估值水平（高/中/低）
- 成长性排名
- 盈利能力排名
- 竞争优势与劣势总结

---

### Step 4：历史调研问题回顾

**4.1 搜索历史调研记录**

使用 `web_search` 查询近期投资者关系活动记录：

```bash
web_search --query "{公司名称} 投资者关系活动记录表 机构调研 问答" --numResults 10
```

**4.2 整理历史问题**

从最近的投资者关系活动记录表（3-5 次）中提取：
- 机构关注的高频问题
- 管理层回复要点
- 未明确回答的遗留问题
- 问题涉及的主要方向（战略/财务/业务/风险）

**4.3 问题分类**

将历史问题按以下类别分类：
- 战略规划类
- 业务发展类
- 财务状况类
- 风险因素类
- 行业竞争类

---

### Step 5：生成调研问题清单

**5.1 问题设计原则**

| 原则 | 说明 |
|------|------|
| **针对性** | 结合公司具体情况，避免通用问题 |
| **层次性** | 从战略到执行，从宏观到微观 |
| **开放性** | 多用"如何"、"为什么"引导深入回答 |
| **可验证** | 问题答案可通过后续数据验证 |
| **差异化** | 避免重复公开信息，关注预期差 |

**5.2 问题数量**

生成**10-12 个**核心调研问题，覆盖以下维度：

| 维度 | 问题数量 |
|------|----------|
| 战略规划 | 2-3 个 |
| 业务发展 | 3-4 个 |
| 财务状况 | 2-3 个 |
| 风险因素 | 1-2 个 |
| 行业竞争 | 1-2 个 |

**5.3 问题质量要求**

每个问题应包含：
- **问题背景**：简要说明提问的原因/依据
- **核心问题**：清晰具体的问题表述
- **追问方向**：可能的后续追问点

---

### Step 6：生成调研大纲报告

**6.1 组织内容结构**

按照下方【报告结构模板】组织报告内容。

**6.2 格式化输出**

1. 使用 Markdown 标题层级（#、##、###）组织章节
2. 使用表格展示财务数据和对比信息
3. 使用列表展示要点和风险提示
4. 确保逻辑清晰、层次分明

**6.3 输出报告**

**将完整的结构化报告直接在对话中展示，无需发送文件。**

## 七、风险提示

1. **信息时效性**：本大纲基于公开信息生成，部分数据可能存在滞后，建议调研前更新最新数据
2. **行业差异**：本大纲为通用模板，特定行业（如银行、保险、地产）需补充行业特有指标
3. **调研礼仪**：避免询问未公开的重大敏感信息，遵守信息披露相关规定
4. **独立判断**：管理层回复可能存在主观倾向，需结合多方信息独立验证

## 特殊场景处理

### 场景一：用户未提供公司名称

当用户只说"生成调研大纲"但未指定公司时：

1. 主动询问用户需要调研哪家上市公司
2. 可提供近期热门调研公司供参考

### 场景二：公司信息较少（次新股/小市值）

当目标公司为次新股或小市值公司，公开信息有限时：

1. 在报告中标注"信息有限"
2. 重点依赖招股书和最新财报
3. 增加"信息获取建议"板块
4. 调研问题中增加基础信息确认类问题

### 场景三：跨行业公司

当目标公司业务跨多个行业时：

1. 分别分析各业务板块
2. 选择各业务板块的可比公司
3. 调研问题覆盖各业务板块

### 场景四：港股/美股上市公司

当目标公司为港股或美股上市公司时：

1. 调整财务指标单位（人民币/美元/港币）
2. 选择相应市场的可比公司
3. 注意不同市场的披露要求差异
4. 使用对应的港股/美股 gildata-aidata 工具

## 注意事项

1. **数据准确性**：财务数据需标注来源和报告期，优先使用 gildata-aidata 获取
2. **客观中立**：分析应基于事实，避免主观臆断
3. **信息时效**：优先使用最新公开信息（最近 12 个月）
4. **合规提示**：提醒用户遵守调研相关规定，不获取内幕信息
5. **问题质量**：调研问题应避免可从公开信息获取的答案
6. **同业选择**：可比公司应选择业务模式最接近的 3-5 家
7. **行业差异**：通用模板可能不适用于所有行业，需提示用户
8. **数据来源**：财务/估值/一致预期优先 gildata-aidata，基本信息/调研记录使用 web_search
9. **简洁优先**：报告控制在适中长度，重点突出
10. **持续更新**：提示用户调研前更新最新数据
11. **输出方式**：**报告内容直接在对话中展示，无需写入文件或发送给用户**
