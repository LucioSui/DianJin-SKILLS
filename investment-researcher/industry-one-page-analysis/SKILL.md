# 行业一页纸分析技能

## 技能定位

本技能面向**投资经理、研究员、基金经理**，提供行业快速分析工具。在一页纸篇幅内，聚焦行业最核心的投资信息，帮助快速判断行业投资价值和风险，适合晨会汇报、快速决策、赛道初筛等场景。

与 `industry-deep-analysis` 的区别：
- **深度分析**：八大维度，全面深入，适合深度研究和首次覆盖
- **一页纸分析**：六大核心维度，精炼简洁，适合快速决策和赛道跟踪

## 触发场景

当用户出现以下任一表述时触发本技能：
- "XX 行业一页纸"
- "一页纸分析 XX 行业"
- "快速分析 XX 行业"
- "XX 行业速评"
- "XX 行业简单分析"
- "XX 行业核心要点"
- "XX 行业投资亮点"
- "XX 行业快速点评"
- "帮我快速看一下 XX 行业"
- "XX 行业怎么样"
- 其他要求对行业进行简洁快速分析的请求

## 数据获取策略

### 核心数据源

**Gildata-AIData (gildata-aidata)** 作为主要数据源，提供全方位的行业金融数据与投研分析工具：

| 数据需求 | 推荐工具 | 说明 |
|---------|---------|------|
| 行业估值指标 | `IndustryValuation` | PE、PB、总市值、流通市值等 |
| 行业行情数据 | `IndustryDailyQuote` / `IndustryRangeQuote` | 日行情、区间行情 |
| 行业资金流向 | `IndustryCapitalFlow` | 主力、散户资金净额 |
| 行业财务指标 | `IndustryFinancialAnalysis` | 盈利能力、偿债能力、成长能力 |
| 行业成分股 | `IndustryConstituentStocks` | 行业成分股列表及权重 |
| 行业舆情资讯 | `IndustryNewslist` | 行业相关新闻、政策动态 |
| 行业研报观点 | `IndustryAnalysisViewpoints` | 券商研报对行业的分析观点 |
| 宏观行业数据 | `MacroIndustryEDB` | 宏观经济与行业经济指标 |
| 板块排序 | `SectorRank` | 板块涨跌幅、换手率等排序 |
| 行业研报 | `ResearchReport` | 按行业筛选券商研报 |

### 数据获取优先级

1. **行业估值与行情**：优先 `IndustryValuation` + `IndustryDailyQuote`
2. **行业财务数据**：优先 `IndustryFinancialAnalysis`
3. **行业舆情与政策**：优先 `IndustryNewslist` + `IndustryAnalysisViewpoints`
4. **宏观行业数据**：优先 `MacroIndustryEDB`
5. **网络搜索作补充**：对于最新政策、市场解读等，使用 `web_search` 补充

## 分析框架（一页纸六大维度）

### 一、行业概况（基本信息）

**数据获取**：
```
使用 gildata-aidata IndustryConstituentStocks 查询：
- 获取行业成分股列表，了解行业覆盖范围

使用 gildata-aidata MacroIndustryEDB 查询：
- 查询行业相关宏观经济数据

使用 web_search 查询：
- "[行业名称] 行业定义 主要产品 应用场景"
```

**核心信息**：
- 行业定义/分类
- 产业链位置
- 主要产品/服务
- 下游应用领域

### 二、市场规模（成长空间）

**数据获取**：
```
使用 gildata-aidata MacroIndustryEDB 查询：
- 查询行业市场规模、产量、销量等经济指标

使用 web_search 查询：
- "[行业名称] 市场规模 2024 2025 2026 增长率"
```

**核心指标**（表格呈现）：
| 指标 | 2023 | 2024 | 2025E | 增速 |
|------|------|------|-------|------|
| 市场规模（亿元） | | | | |
| 同比增速（%） | | | | |
| 市场渗透率（%） | | | | |

### 三、竞争格局（行业集中度）

**数据获取**：
```
使用 gildata-aidata IndustryConstituentStocks 查询：
- 获取行业成分股及权重，识别龙头企业

使用 gildata-aidata SectorRank 查询：
- 查询板块内个股排序，了解市值、营收分布

使用 web_search 查询：
- "[行业名称] CR3 CR5 市场集中度 主要企业"
```

**核心信息**：
- 行业集中度（CR3/CR5）
- 主要龙头企业及市占率
- 竞争态势（分散/集中）
- 进入壁垒

### 四、行业趋势（发展方向）

**数据获取**：
```
使用 gildata-aidata IndustryNewslist 查询：
- 查询行业最新舆情、政策动态

使用 gildata-aidata IndustryAnalysisViewpoints 查询：
- 获取券商研报对行业趋势的分析观点

使用 web_search 查询：
- "[行业名称] 发展趋势 2025 2026 技术革新"
```

**分析要点**：
- 技术发展趋势
- 政策导向
- 需求变化趋势
- 供给格局变化

### 五、投资机会（投资逻辑）

**数据获取**：
```
使用 gildata-aidata IndustryValuation 查询：
- 获取行业 PE、PB 等估值指标及历史分位

使用 gildata-aidata IndustryCapitalFlow 查询：
- 获取行业资金流向，判断资金态度

使用 gildata-aidata ResearchReport 查询：
- 按行业筛选最新研报，获取投资逻辑

使用 web_search 查询：
- "[行业名称] 投资机会 细分赛道 2025 2026"
```

**核心信息**：
- 核心投资逻辑（1-3 点）
- 重点细分赛道
- 近期催化剂
- 估值水平

### 六、主要风险（风险提示）

**数据获取**：
```
使用 gildata-aidata IndustryNewslist 查询：
- 查询行业负面舆情、风险事件

使用 web_search 查询：
- "[行业名称] 风险因素 政策风险 市场风险"
```

**风险要点**：
- 列出 2-4 个核心风险点
- 按重要性排序

## 注意事项

1. **篇幅控制**
   - 严格控制在"一页纸"篇幅内
   - 避免冗长描述，每点 1-2 句话
   - 优先使用表格呈现数据

2. **数据时效性**
   - 使用最新可得的行业数据
   - 标注数据截止日期或预测年份
   - 区分历史数据和预测数据

3. **聚焦核心**
   - 只呈现最关键的信息
   - 避免细节堆砌
   - 突出投资逻辑主线

4. **客观中立**
   - 保持客观，避免过度推荐
   - 风险提示必须充分
   - 明确说明"不构成投资建议"

5. **报告交付要求**
   - **直接在对话中显示完整报告内容**
   - 不需要发送文件
   - 使用 Markdown 格式，确保可读性

## 特殊情况处理

### 情况 1：数据获取失败
- 尝试使用替代数据源
- 如仍无法获取，标注"数据暂缺"
- 不编造数据

### 情况 2：行业范围模糊
- 如用户只说"新能源行业"（范围太宽）
- 主动询问具体细分行业（如光伏、风电、锂电等）
- 或选择一个主流细分领域进行分析并说明

### 情况 3：新兴行业数据不足
- 说明数据局限性
- 使用 web_search 获取最新信息
- 在报告中说明数据覆盖范围

### 情况 4：用户要求更详细
- 如用户表示"太简单了，要更详细"
- 建议调用 `industry-deep-analysis` 技能进行深度分析
- 或在一页纸框架内适当扩展各部分内容
